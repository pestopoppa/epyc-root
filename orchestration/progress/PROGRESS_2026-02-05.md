# Orchestration Progress: 2026-02-05

## Reward Injection Event Loop Fix + Replay

### Summary

Fixed a blocking bug in `/chat/reward` that caused reward injection timeouts during 3-way evaluation. The async endpoint was calling synchronous code (HTTP embedding + FAISS + SQLite) on the event loop, blocking all concurrent requests. 16/44 evaluated questions had incomplete reward injection. Created a replay script to backfill from checkpoint data — 51/51 rewards recovered.

### Problem

During 3-way eval, reward POST calls to `http://localhost:8000/chat/reward` timed out at 10s. Two failure patterns:

| Pattern | Trigger | Result |
|---------|---------|--------|
| Sporadic | Embedder slow under load | 1 reward lost per question |
| Cascading | After architect_coding 600s timeout | All rewards lost (0 injected) |

Root cause: `store_external_reward()` blocks the uvicorn event loop (single worker, sync HTTP + FAISS + SQLite).

### Changes

| File | Change |
|------|--------|
| `src/api/routes/chat.py` | `asyncio.to_thread(store_external_reward, ...)` — offload to thread |
| `scripts/benchmark/replay_missing_rewards.py` | NEW — scan checkpoints, re-inject missing rewards (30s timeout) |

### Impact

- **Before**: 22/51 rewards lost across 16 questions (43% failure rate on affected questions)
- **After**: 51/51 replayed successfully, 0 timeouts with thread-offloaded endpoint

### Test Status

- 2954 passed, 65 skipped, 0 failures

## Embedding Caching for Reward Injection

### Summary

Added embedding precomputation to eliminate redundant embedding calls during 3-way reward injection. Each question triggers 3 reward injections (SELF:direct, SELF:repl, ARCHITECT) with the same `task_description`. Previously, each injection recomputed the embedding, causing ~3.5s per injection and occasional 10s timeouts. Now the embedding is computed once and reused.

### Problem

Log showed intermittent timeouts despite 6 parallel embedders:
```
reward[ARCHITECT] = 1.0
reward[SELF:direct] = 1.0
reward[SELF:repl] = 1.0
Reward injection error for SELF:direct: timed out
Injected 2 rewards
```

Root cause: Each `/chat/reward` call embeds the same `prompt[:200]` text. With 3 sequential injections at ~3.5s each, one could exceed the 10s timeout.

### Changes

| File | Change |
|------|--------|
| `src/api/models/requests.py` | Added `embedding: list[float] | None` to `RewardRequest` |
| `src/api/routes/chat.py` | Pass `request.embedding` through to `store_external_reward()` |
| `src/api/services/memrl.py` | Accept and forward `embedding` parameter |
| `orchestration/repl_memory/q_scorer.py` | `score_external_result()` uses precomputed embedding if provided |
| `scripts/benchmark/seed_specialist_routing.py` | Added `_precompute_embedding()`, cache & reuse for all 3 injections |
| `tests/unit/test_memrl_service.py` | Updated `StubQScorer` signature |

### Impact

- **Before**: 3 embedding calls per question (~10.5s total), occasional timeouts
- **After**: 1 embedding call per question (~3.5s total), no timeouts expected

### Test Status

- All related tests pass (45 tests in test_memrl_service.py + test_api.py)
