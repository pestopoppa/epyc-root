## Seeding Infra and Vision Routing Follow-up (late session)

### Summary

Focused on 3-way seeding failures where calls returned `INFRA` with `0 tok` on heavy ports. The goal was remediation (models generating tokens), not just faster skipping.

### Root Causes Confirmed

1. Heavy-port pre-response stalls on architect ports (`8083/8084`) produced exact timeout-bound failures (`420s/540s`) with zero tokens.
2. Slot erase endpoint compatibility mismatch: some backends rejected `POST /slots/{id}?action=erase` (HTTP 501), so stale work could remain in-flight.
3. Delegation runtime bug in chat pipeline: `'ExecutionResult' object has no attribute 'get'` forced fallback behavior and inflated contention.
4. Forced-role quality escalation could role-hop in eval path, confounding `force_role` integrity.

### Remediations Applied

| Area | Change |
|------|--------|
| Seeding timeouts | Adaptive per-call timeout with observed-runtime bumping (repl from direct, architect from max(self legs)) |
| Heavy path gating | Pre-call busy-port check + recovery path + one retry after recovery on zero-token infra |
| Slot cleanup | Slot erase now probes multiple API styles (`POST_QUERY`, `GET_QUERY`, `POST_JSON`) before disabling per port |
| Delegation bug | Replaced dict-style `ExecutionResult` access with dataclass fields in delegation/stages |
| Forced-role integrity | Disabled forced-run role hopping while preserving delegation where allowed |
| MemRL pressure | Skip async MemRL task scoring for `force_role && real_mode` benchmark calls |

## Inference Tap Stability Patch (late session 2)

### Summary

Reduced orchestration stress from TUI live output by changing tap behavior from "always force streaming" to policy-based streaming.

### Changes

| Area | Change |
|------|--------|
| Tap stream policy | Added `INFERENCE_TAP_STREAM_MODE` with `safe` (default), `force`, `off` |
| Heavy roles | In `safe` mode, heavy roles no longer force SSE streaming transport (`architect_*`, `vision_*`, `worker_vision`, `ingest_long_context`) |
| Tap writes | Switched to persistent per-section file handle (close on section exit) instead of open/close per chunk |
| Non-stream path tap output | When tap is active but streaming is disabled for role, full response + timings are still written to tap |

### Why

Previous tap behavior changed transport mode for all calls whenever tap was active. Under lock-heavy roles this increased long lock holds and timeout risk. New policy preserves near-real-time output on safer roles while avoiding unnecessary instability on heavy roles.
