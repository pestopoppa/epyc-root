# Progress Report — 2026-03-09

## Remove KV Cache Quantization for Architect Models

### Change
Removed `_KV_QUANTIZED_ROLES` and associated `--cache-type-k q8_0` flag from `orchestrator_stack.py`. Both architect models (`architect_general` Qwen3-235B-A22B and `architect_coding` Qwen3-Coder-480B-A35B) now use f16 for K and V cache, matching all other roles.

### Rationale
With 1TB+ system RAM, the VRAM savings from q8_0 K-cache quantization are unnecessary. K-cache is more sensitive to quantization than V-cache — precision loss in attention logits can affect expert routing in large MoE models and degrade long-context recall. Removing it eliminates any potential quality risk.

### Expected Impact
- **Quality**: Removes any precision loss from K-cache quantization on the two largest models
- **Speed**: ~2-5% slower generation (larger KV reads during attention), negligible with available memory bandwidth
- **Memory**: ~2x KV cache memory for architects (acceptable given 1TB+ headroom)
