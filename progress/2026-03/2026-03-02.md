### Phase 1: Infrastructure Stability

#### WI-1: frontdoor:direct Timeout/Retry Fix
**File:** `epyc-orchestrator/scripts/benchmark/seeding_eval.py`

**Problem:** 49% infra error rate on frontdoor:direct (339/689). HTTP 504 Gateway Timeout contained "timeout" in error string, which triggered the skip-retry logic designed for client socket timeouts. All 504s are server-side (retryable) but were treated as client-side (non-retryable).

**Fix:**
- Distinguished HTTP 5xx (`"server error '" in error_msg`) from client socket timeouts (`ReadTimeout`, `ConnectTimeout`)
- Added retry-after-recovery for frontdoor:direct after HTTP 5xx, matching existing architect retry behavior
- Only true client socket timeouts now skip retry

### Phase 2: Prompt Engineering

#### WI-2 + WI-4: REPL Tool Suppression Fix
**Files:** `epyc-orchestrator/src/prompts/root_lm_system.txt`, `src/prompt_builders/constants.py`

**Problem:** 93% of REPL runs had zero tool use. The prompt said "call FINAL() immediately, no code needed" and tool descriptions said "Do NOT use for general knowledge." simpleqa REPL had 0% pass rate despite web_search being available.

**Fix:**
- Rewrote opening to tool-forward framing: "You have powerful tools — use them."
- Reordered COMPACT_ROOT_LM_TOOLS: web_search, search_wikipedia, run_python_code now top 3
- Flipped DEFAULT_ROOT_LM_RULES: tool use guidance first, direct answers demoted
- Added: "When in doubt, search first — a wrong answer from memory is worse than a slower correct answer from search"
- Removed "Do NOT use for general knowledge" from web_search/search_wikipedia descriptions
- Added competitive programming examples with run_python_code testing

#### WI-3 + WI-5: Architect Protocol Compliance
**Files:** `epyc-orchestrator/src/prompts/architect_system.txt`, `src/api/routes/chat_delegation.py`

**Problem:** 89 format_violation records. Architect prompt showed JSON examples (`{decision:"Redis",why:[...]}`) but parser expected `D|answer` or `I|brief:...|to:role` prefixes. 87 misrouted_to_coder records where non-coding tasks were delegated to coder_escalation.

**Fix (prompt):**
- Complete rewrite with 6 positive D|/I| examples and 3 negative examples
- Added DELEGATION RULES section: factual → D| direct, coding → coder_escalation, research → worker_explore
- Added valid delegation targets list (worker_general, worker_explore, worker_summarize, worker_math, coder_escalation)
- Reasoning goes in `<think>` tags, then D| or I| on its own line

**Fix (parser):**
- Added preamble-stripping: if response doesn't start with D|/I|, search for it mid-response
- Logs recovery events for monitoring

### Phase 3: Polish & Monitoring

#### WI-6: function_repr_leak Guard
**File:** `epyc-orchestrator/src/api/routes/chat_utils.py`

**Problem:** 5 records where answer was `<function foo at 0x...>`. Existing callable guard in `_final()` handles when model passes callable to FINAL(). But when model never calls FINAL() and REPL captures bare expression evaluation, the repr leaks through `_resolve_answer()`.

**Fix:** Added regex guard in `_resolve_answer()` that strips `<function|class|module ... at 0x...>` repr strings from both captured output and final answer.

#### WI-9: New Anomaly Detectors
**File:** `epyc-orchestrator/src/pipeline_monitor/anomaly.py`

Added 5 new signal detectors wired into `compute_anomaly_signals()`:
- `timeout_no_retry` (0.5): infra timeout where retry was skipped
- `max_turns_exhausted` (1.0): REPL hit max turns without usable answer
- `tool_discovery_missing` (0.3): REPL mode with zero registered tool calls
- `malformed_delegation` (0.5): has |to: but no I|brief: prefix
- `coder_on_knowledge_task` (0.5): coder received f1/exact_match question

New backward-compatible parameters: `error_type`, `tools_called`.

#### WI-7, WI-8: Resolved by Earlier Phases

#### Critical Discovery: Prompt File Mismatch

Phase 2 edits to `src/prompts/root_lm_system.txt` and `src/prompts/architect_system.txt` were **NOT being used by the running API**. The API reads from `orchestration/prompts/*.md` via `resolve_prompt()` in `resolver.py` — these `.md` files are hot-swappable (read on every request). The `src/prompts/*.txt` files are fallback constants only.

Additionally, `COMPACT_ROOT_LM_TOOLS` is only used for MINIMAL prompt style, but the default is STRUCTURED, which uses `DEFAULT_ROOT_LM_TOOLS`.

**Corrective actions:**
1. Updated `orchestration/prompts/root_lm_system.md` with tool-forward system prompt
2. Added "### Critical Tools — USE THESE FIRST" section to `DEFAULT_ROOT_LM_TOOLS` in `constants.py`
3. Confirmed `orchestration/prompts/architect_investigate.md` already had correct D|/I| format
