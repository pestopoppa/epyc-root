## Research Intake: OpenDataLoader PDF (intake-161)

### Summary
Processed [OpenDataLoader PDF](https://github.com/opendataloader-project/opendataloader-pdf) (v2.0.0, Apache 2.0) through research intake pipeline. Initially assessed as irrelevant, corrected after identifying the orchestrator's full document processing pipeline (51 files across `src/services/`, `src/vision/`).

### Key Findings

**What it is**: Java-based PDF parser with Python SDK. Two modes:
- Local: rule-based XY-Cut++ algorithm, 0.05s/page, 0.72 overall accuracy
- Hybrid: routes complex pages to AI backend, 0.43s/page, 0.90 overall, 0.93 table accuracy

**Benchmark results** (opendataloader-bench, MIT, 200 PDFs):

| Engine | Overall | Reading Order | Table | Heading | Speed |
|--------|---------|---------------|-------|---------|-------|
| ODL hybrid | **0.90** | **0.94** | **0.93** | **0.83** | 0.43s |
| docling | 0.86 | 0.90 | 0.89 | 0.80 | 0.73s |
| ODL local | 0.84 | 0.91 | 0.49 | 0.76 | **0.05s** |

**XY-Cut++ paper**: [arXiv:2504.10258](https://arxiv.org/abs/2504.10258) — hierarchical mask mechanism, 98.8 BLEU, +24% over baselines on complex layouts.

### Current Pipeline Gaps Identified
1. No table extraction (pdftotext mangles tables)
2. No reading order (pdftotext `-layout` interleaves multi-column)
3. Binary routing (pdftotext OR LightOnOCR, no per-page complexity)
4. Blind figure analysis (VL models get cropped images without document context)
5. No prompt injection filtering

### Integration Strategy (Three-Phase)

**Phase 1**: Replace pdftotext with ODL local (same speed, better structure)
**Phase 2**: Structured context for downstream models — biggest win:
- VL models receive semantic type + caption + surrounding text + heading position
- Chunker uses heading hierarchy instead of regex
- Tables routed to ODL hybrid for 0.93 accuracy
**Phase 3**: Hybrid sidecar + benchmark integration
- Test LightOnOCR-2-1B as ODL hybrid backend (replace docling-fast)
- Add opendataloader-bench 200 PDFs to our benchmark suite
- Three-way routing: ODL local → ODL hybrid → LightOnOCR
