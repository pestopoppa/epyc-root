## Bug Fixes During Continuation

1. **`features().factual_risk_mode` AttributeError** — routing.py:405 referenced a non-existent field on the Features dataclass. Fixed: replaced with `get_mode()` from `factual_risk.py`. Would have crashed on any `enforce` mode activation.

2. **No env var override for factual risk mode** — `get_mode()` in `factual_risk.py` only read from `classifier_config.yaml`. Added `ORCHESTRATOR_FACTUAL_RISK_MODE` env var override (precedence over YAML). Enables A/B testing without YAML edits.

3. **Health probe `full:` prefix bug** — API health endpoint reports "degraded" because `backend_probes` try to hit raw `full:http://...` URL strings. The `full:` prefix (ConcurrencyAwareBackend convention) isn't stripped. Logged as LC-5 in routing index (fix later).

## RI-7: A/B Test — COMPLETE (underpowered)

70 questions per arm (10/suite × 7 suites).

| Action | Arm A (off) | Arm B (enforce) | Delta | p-value |
|--------|-------------|-----------------|-------|---------|
| SELF:direct | 61.4% | 62.9% | +1.4pp | 0.86 (NS) |

**Verdict**: All differences not statistically significant. Enforce mode neither helps nor hurts. Test underpowered (70 vs target 500/arm) — re-run with larger sample recommended before production rollout (RI-10).

Tool value worsened slightly under enforce (helped/hurt = 5/19 vs 9/18) but not significant.
