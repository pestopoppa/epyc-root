## Standalone Seeding Watch TUI

Created `scripts/benchmark/watch_seeding.py` in epyc-orchestrator — a standalone TUI that observes an actively running seeding session from a separate terminal.

### Problem

The existing `seeding_tui.py` is in-process only (activated via `--tui` on `seed_specialist_routing.py`). No way to observe a running seeding session from another terminal/agent.

### Solution

Passive observer TUI that reuses `TapTailer` and styling from `seeding_tui.py`:
- Writes the sentinel file to activate inference tap output in the running orchestrator
- Tails inference tap + REPL tap files via `TapTailer`
- Auto-discovers most recent checkpoint JSONL for live pass/fail stats with per-suite bar chart
- Cleans up sentinel on Ctrl-C exit

### Bug Fix: TapTailer Cross-Process Read Stale

Fixed a `tail -f` buffering bug in `TapTailer._run()` (`seeding_tui.py`): Python's text-mode `BufferedReader` caches EOF state after `fh.read()` returns `""`. Subsequent reads return `""` without a syscall, missing cross-process appends.

Fix: switched to binary mode (`"rb"`) + explicit `fh.seek(fh.tell())` before each read to invalidate the buffer. This affects both the in-process TUI and the new watch TUI (they share `TapTailer`).

1. **Draft models are nearly useless**: Sub-2B models that had 60-100% lazy scores actually score 0-20%. Universal failures: prompt echoing, degenerate repetition loops, token limit exhaustion.
