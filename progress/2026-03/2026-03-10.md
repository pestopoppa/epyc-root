### SSM Checkpoint/Restore Overhead Analysis

Investigated `llama-memory-recurrent.cpp` checkpoint/restore implementation. Per speculation round on hybrid models:

1. `checkpoint()` — `ggml_backend_tensor_get` for ALL r_l + s_l tensors (GPU→CPU, ~2GB for Qwen3.5)
2. N draft decodes
3. 1 target verification decode
4. `restore()` — `ggml_backend_tensor_set` for ALL tensors (CPU→GPU, ~2GB)
5. Re-advance decode — extra `llama_decode()` to replay accepted tokens through recurrent state

**Optimization opportunities:**
1. GPU-to-GPU copy (avoid CPU roundtrip) — use `ggml_backend_tensor_copy`
2. Lazy restore — skip when all drafts accepted
3. Partial checkpoint — only save active cells (2-8 vs 1024) and affected layers
4. Async transfers — pipeline save with draft generation

## SSM Checkpoint Optimization — Double-Buffer Pointer Swap

Implemented double-buffer checkpoint/restore optimization on `feature/ssm-checkpoint-opt` branch (from `production-consolidated-v2`).

### What Changed

**Files**: `llama-memory-recurrent.h`, `llama-memory-recurrent.cpp`

1. **Shadow tensor allocation**: Pre-allocate a second set of r_l/s_l tensors at construction time (same backend as originals). Doubles recurrent memory but enables O(1) restore.

2. **checkpoint()**: Uses `ggml_backend_tensor_copy()` (device-to-device copy) to save active→shadow tensors. Eliminates CPU roundtrip. Partial cell metadata — only saves non-empty cells (typically 1-8 vs 1024 total).

3. **restore()**: `std::swap(r_l, shadow_r_l)` + `std::swap(s_l, shadow_s_l)` — O(1) pointer swap instead of memcpy. Shadow tensors (containing checkpointed state) become the active set instantly.

### Expected Impact

- **Restore**: O(1) pointer swap vs ~144MB memcpy → near-zero cost
- **Checkpoint**: Same cost (memcpy still needed to save state, but direct tensor-to-tensor)
- **Memory**: 2x recurrent state allocation (shadow buffers)
- **Target**: Eliminates ~50% of checkpoint/restore overhead for partial-accept speculation rounds

## Freeze-Recurrent Speculation — Implementation

Radical optimization: skip SSM state writes entirely during speculation. Recurrent layers still compute (read state, produce output) but don't write updated state back to cache. Eliminates all three SSM overhead costs:
- Checkpoint save (eliminated — nothing to save)
- Restore memcpy (eliminated — nothing to restore)
- Re-advance decode (replaced with advance-only for accepted tokens, same as checkpoint path)

### Implementation

**New flag**: `--freeze-recurrent-draft` (env: `LLAMA_ARG_FREEZE_RECURRENT_DRAFT`)

**Runtime API**: `llama_set_freeze_recurrent(ctx, bool)` — sets `cparams.freeze_recurrent`

**Model changes** — conditional `ggml_build_forward_expand` for state writes:
- `src/models/qwen35.cpp` — 2 state write points guarded
- `src/models/qwen35moe.cpp` — 2 state write points guarded (Qwen3.5 MoE models)
- `src/models/qwen3next.cpp` — 2 state write points guarded (Qwen3-Next-80B)

**Server integration** (`server-context.cpp`):
- When `freeze_recurrent_draft` enabled + model has recurrent memory:
  - Pre-speculation: `llama_set_freeze_recurrent(ctx, true)` instead of checkpoint
  - Post-speculation: `llama_set_freeze_recurrent(ctx, false)`, advance state through accepted tokens
  - Checkpoint save/restore entirely skipped

**Trade-off**: Later draft tokens use stale SSM state → lower acceptance rate. Net throughput depends on whether eliminated overhead outweighs acceptance drop.
