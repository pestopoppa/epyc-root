## Phase 2 — DySpec Dynamic Tree Construction (Complete)

Replaced Phase 1's per-depth expansion with heap-based greedy expansion (DySpec algorithm from handoff).

### Key Changes to `common_speculative_state_tree::draft()`

- **Priority queue** (`std::priority_queue<frontier_node>`): nodes keyed by cumulative log probability
- **Budget termination**: tree grows until `n_max` nodes (was depth-limited)
- **Global priority**: most promising nodes expanded first regardless of depth
- Cap at 32 total seq_ids, decreasing branching factor with depth

## Phase 2.5 — Runtime Debugging & Fixes

Three critical runtime bugs discovered and fixed during validation:

### Bug 1: Draft context `n_seq_max=1`
- **Symptom**: `GGML_ASSERT(logits != nullptr)` in `common_sampler_sample`
- **Root cause**: Draft context created with `n_seq_max=1` (default), can't support tree branching
- **Fix**: Set `cparams_dft.n_seq_max = 33` when `p_split > 0`

### Bug 2: Non-unified KV cache `seq_cp` failure
- **Symptom**: `GGML_ASSERT(is_full && "seq_cp() is only supported for full KV buffers")`
- **Root cause**: Cross-stream seq_cp with bounded position range rejected by KV cache
- **Fix**: Set `cparams_dft.kv_unified = true` + use `p1=-1` (full copy) for all `llama_memory_seq_cp`

### Bug 3: KV/prompt_dft mismatch between draft rounds
- **Symptom**: `decode: failed to initialize batch` on second speculation round
- **Root cause**: After tree expansion, best path tokens appended to `prompt_dft` but seq 0 KV has primary path. Mismatch causes position consistency check failure.
- **Fix**: After tree construction, trim seq 0 KV back to pre-tree state (`n_past+1`), remove all non-zero seq_ids. Don't append tree tokens to `prompt_dft`. Each draft round rebuilds tree fresh.

### Multi-path target verification (deferred to Phase 3)
- Target context has `n_seq_max=1` (from `-np 1`), can't support tree seq_ids
- Bumping n_parallel splits context into streams (reduces per-slot n_ctx)
- Requires `kv_unified=true` on target context — needs `common_params.kv_unified` field or manual context creation
- **Current state**: Multi-path verification code present but disabled (`if (false && ...)`)
- Tree still provides benefit through best-path selection during drafting

## Phase 3 — Multi-Path Target Verification (Complete)

Enabled multi-path tree verification on target context:
- Added `int32_t n_seq_max` override to `common_params` (0 = use n_parallel)
- Target context: `kv_unified=true` + `n_seq_max = 9 * n_parallel` when user sets `--kv-unified`
- Fixed KV copy bug: position-bounded `seq_cp(pos_sampled, -1)` instead of full range
- Disabled multi-path verification for hybrid/recurrent models — fundamentally incompatible (SSM state can't fork)
- Alternative paths winning consistently (14-20 wins per 128-256 token generation)

### Hybrid Model Incompatibility
Multi-path target verification requires `llama_memory_seq_cp` on the target context, which breaks recurrent state management on hybrid models (Qwen3.5 = Mamba2 + attention). The SSM state is sequential — you can't fork it across alternative verification paths without re-advancing through the full sequence for each path. Dense models don't have this problem.

1. `common/common.h` — `n_seq_max` field
2. `common/common.cpp` — `common_context_params_to_llama` uses override
3. `tools/server/server-context.cpp` — target context config (gated on `--kv-unified`), multi-path verification (gated on `!has_recurrent`)

| Config | t/s | Accept% | Acc/Tot | Delta |
|--------|-----|---------|---------|-------|
| Linear | 5.75 | 51.2% | 292/570 | base |
| p_split=0.05 | 4.26 | 32.1% | 232/723 | -26% |

Without multi-path verification, tree's `get_best_path()` (optimized for draft model probability) often disagrees with target more than greedy path would.

### Root Cause: `get_best_path()` vs Greedy

**Critical finding**: `get_best_path()` returns the path with highest cumulative log probability on the DRAFT model. This is NOT the same as the greedy path (top-1 at each step). When they differ, the draft-optimal path often has WORSE target agreement than greedy. This is why tree acceptance rate (74.5%) is lower than linear (77.3%) on 32B — path 0 is draft-model-optimal, not target-optimal.

**Fix needed**: Return the greedy path as `slot.drafted` (path 0), and use tree alternatives for multi-path verification. This guarantees path 0 is never worse than linear — tree can only improve.

## Greedy Path Fix (Complete)

Added `get_greedy_path()` method to `speculation_tree` — follows primary children (node 0 → first child at each depth) instead of highest cumulative log probability leaf. Changed `draft()` to return `tree.get_greedy_path()` instead of `tree.get_best_path()`.

### Impact: Minimal

Benchmarking showed `get_best_path()` and `get_greedy_path()` return the same path in almost all cases on the 32B pair — the draft model's greedy path IS usually the highest-cumulative-probability path. Before fix: 374/502 (74.5%), after: 375/503 (74.6%).

### Conclusion

The path selection was not the bottleneck — tree construction overhead is. Viable optimization paths:
- Batch sampler cloning (currently per-node, could batch)
- Lazy KV copies (defer until verification, not during tree building)
- Reduce branching factor (fewer alt paths = less overhead)
- f16 targets (near-flat verification cost → overhead amortized better)
