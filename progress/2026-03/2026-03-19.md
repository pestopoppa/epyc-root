# 2026-03-19 Progress

## Multi-Model Page Cache Optimization — S1 Baseline (Session 2026-03-19a)

### Context
All inference acceleration benchmarks complete (session 2026-03-18d). New work: page cache optimization for multi-model concurrent serving. Research intake of flash-moe surfaced insight about OS page cache management for mmap'd models.

### S1: Baseline Page Residency Measurement
Loading 3 production models sequentially (480B→235B→30B) to measure:
- Page residency (RSS/file size %) after each model load
- Cold vs warm request latency
- Page fault rates during inference bursts

### S1 Results: Page Cache Contention is REAL

**Cold vs Warm Latency:**

| Model | Cold (1st) | Warm (2nd) | Warm (3rd) | Cold/Warm Ratio |
|-------|-----------|-----------|-----------|----------------|
| **480B** | **185,271 ms** (3 min!) | 4,299 ms | 8 ms | **>23,000x** |
| 235B | 7,276 ms | 2,667 ms | 1,894 ms | 3.8x |
| 30B | 7,897 ms | 815 ms | 348 ms | 22.7x |

**Page Residency Tracking:**

| Stage | 480B RSS | 235B RSS | 30B RSS | System Used |
|-------|---------|---------|---------|-------------|
| After 480B load | 311.5 GB | — | — | 461 GB |
| After 235B load | 455.1 GB | 195.7 GB | — | 801 GB |
| After 30B load | **417.7 GB** (-37) | 195.7 GB | 83.1 GB | 867 GB |
| After requests | 441.7 GB (+24) | 180.4 GB (-15) | 83.2 GB | 895 GB |

**Key findings:**
1. **480B cold-start = 185 seconds** — page faults from ~37 GB evicted pages being read back from NVMe
2. 480B lost 37 GB of RSS when 30B loaded (page cache eviction under memory pressure)
3. System at 867 GB / 1.1 TB after all loads — only 5.4 GB free → severe page cache pressure
4. Warm latency converges to <10 ms after 2-3 requests (pages fully resident)
5. perf page faults blocked in container (no permission)

### S3 Results: Page-In Verification

Page-in timing: 480B = 26s (252 GB), 235B = 19s (143 GB).
Post-page-in 480B: 12.7s (14.5x better than S1's 185s). But paging in 480B evicted 235B pages — zero-sum at current memory pressure.

### S2 Results: mlock CONFIRMED WORKING (Session 2026-03-19b)

Container rebuilt with `--ulimit memlock=-1:-1` + privileged mode. mlock now works.

**Setup**: Frontdoor (30B) loaded FIRST with `--mlock`, then 480B (250 GB) + 235B (130 GB) without mlock.

| Phase | Frontdoor Latency | VmLck | RSS |
|-------|------------------|-------|-----|
| warm-baseline (alone) | 264 ms | 17.60 GB | 83.10 GB |
| post-480B (250 GB loaded) | 254-266 ms | 17.60 GB | 83.16 GB |
| post-all (380 GB loaded) | **250-261 ms** | 17.60 GB | 83.16 GB |
| S1 reference (no mlock) | 7,897 ms cold | 0.00 GB | lost pages |

**Result**: mlock completely prevents page eviction. Frontdoor latency: **250ms stable** vs **7,900ms without mlock** (30x improvement). Not zero-sum unlike page-in verification.

Unlocked models still show cold-start (480B: 9s, 235B: 8.9s) confirming mlock is the differentiator, not load order.

### S4 Results: NUMA Binding — No Difference (Session 2026-03-19b)

Tested 3 memory binding strategies (50 requests each, 128 tokens, 235B on node1 as pressure):

| Config | P50 | P95 | P99 | NUMA placement |
|--------|-----|-----|-----|---------------|
| taskset only (first-touch) | 922ms | 1091ms | 4415ms | N0=83.11, N1=0.01 GB |
| numactl --preferred=0 | 936ms | 1101ms | 4360ms | identical |
| numactl --membind=0 | 912ms | 1075ms | 4285ms | identical |

All within noise (~2-3%). First-touch already places all pages on local node when CPUs are pinned. No benefit from explicit memory binding for single-node models.

### Conclusions (Final)

Page cache contention is real and measurable. Root cause: ~508 GB total model footprint with ~1.1 TB RAM. Mitigations (final priority):
1. **`--mlock` for latency-critical models** (CONFIRMED) — lock frontdoor (17.5 GB) + ingest (~46 GB) = 63.5 GB. Requires `ulimit -l unlimited`. **Strongest mitigation: 30x improvement, not zero-sum.**
2. Load largest models first (smallest gets evicted least)
3. Page-in verification as health check (26s cost for 480B, one-time) — use for unlocked models
4. NUMA binding: **taskset alone is sufficient** — no need for numactl --membind
5. Consider replacing 480B (3.4 t/s) with 397B hybrid (12.4 t/s, saves 45 GB)
