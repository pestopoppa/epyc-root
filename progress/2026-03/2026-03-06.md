## Prompt Lookup (`--lookup`) Re-ported to llama.cpp production-consolidated-v2

### Problem

The `--lookup` flag (commit `8e35dbc01` on `production-consolidated`) was lost during the v1→v2 rebase. This blocked all lookup-based benchmark configs (`lookup`, `moe_lookup`, `spec_lookup`, `moe_spec_lookup`).

### Solution

Manual port across 5 files (88 insertions). Cherry-pick failed with 10 conflicts — manual port was cleaner.

| File | Changes |
|------|---------|
| `common/arg.cpp` | `--lookup` CLI flag definition |
| `common/common.h` | `bool lookup` in `common_params` |
| `tools/server/server-task.h` | `bool lookup` in `task_params` |
| `tools/server/server-task.cpp` | JSON serialization (2 `to_json` blocks), defaults, `json_value` parsing |
| `tools/server/server-context.cpp` | `ngram_cache_context` + `lookup_enabled` in `server_slot`; cache clear in `reset()`; `can_speculate()` gate; slot init from task params; spec-first/lookup-fallback draft generation (~70 lines); ngram cache update after normal decode; cache init from prompt tokens; cache update after speculative acceptance |

### Architecture

- **Spec-first, lookup-fallback**: When both `-md` and `--lookup` are active, draft model is tried first. If no draft model loaded, prompt lookup generates draft tokens from ngram cache.
- **Per-slot isolation**: Each slot has its own `ngram_cache_context`, enabling parallel request support.
- **Cache lifecycle**: Built from prompt tokens at `SLOT_STATE_GENERATING`, updated incrementally after each decode and after speculative token acceptance.
