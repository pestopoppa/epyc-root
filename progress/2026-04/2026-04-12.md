### Problems

1. **Dynamic Stack Phase E** (DS-6/DS-7) had no implementation ŌĆö design was complete (gap resolutions 2026-04-09) but no code existed for the QuarterScheduler or stack templates.

2. **AR-3 run 3 was stuck** ŌĆö 128 trials, 93% failure rate. All recent trials had `speed=0` and were rejected by the safety gate's throughput floor (10.2 t/s). Investigation revealed `total_tokens_generated` in `LLMPrimitives` was initialized to 0 but **never incremented** after inference calls, so the eval tower always computed `speed=0`.

### Root Cause (AR-3 speed=0)

`LLMPrimitives.total_tokens_generated` (line 130 of `src/llm_primitives/primitives.py`) was set to 0 at init. The three inference paths (`_llm_call_impl`, and two `llm_batch` variants) tracked `completion_tokens` in `self._current_query.completion_tokens` but never accumulated them into `self.total_tokens_generated`. The response pipeline reads `primitives.total_tokens_generated` to populate the API's `tokens_generated` field, which the eval tower uses to compute `speed = tokens / elapsed`.
