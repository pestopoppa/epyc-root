# 2026-04-05c — Effective Throughput: Tool Tokens as Throughput Amplifiers

## Session Goal

Count tool output tokens and delegate tokens in blended throughput rate. Tools generate tokens the model would have produced — they're throughput amplifiers, not overhead. Without this, autopilot cannot assess what actually makes inference faster.

## Completed

### Effective Throughput Metric

**Concept**: `effective_tps = (model_tokens + tool_output_tokens + delegate_tokens) / wall_clock`

- Tool output tokens estimated as `len(result) // 4` (~4 chars/token), same heuristic as `ExplorationTracker`
- When no tools/delegation: display collapses to standard `t/s` (backward compatible)
- When effective > model: shows `18.6 eff t/s (7.1 model)` and `2041 eff tok (783 model)`

### Data Flow

```
llama-server tokens_predicted → InferenceResult.tokens_generated → ChatResponse.tokens_generated (model tokens)
ToolInvocation.result → len(result)//4 → ToolTiming.output_tokens → ChatResponse.tool_output_tokens
delegation_events[].tokens_generated → delegate tokens
All three → compute_effective_tps() → eval log display
```

### Test Results

7 inline tests passed covering all display modes:
- Model-only: collapses to `7.1 t/s` (no visual change)
