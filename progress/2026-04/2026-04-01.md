# 2026-04-01 — AR-3 First Live Run: Debug & Fix Session

## Summary

AR-3 (first live autoresearch run) launched but all trials failed due to cascading wiring bugs. Spent the session diagnosing, fixing, and hardening the autopilot infrastructure. Program.md completely rewritten to expand autopilot autonomy. Ready for clean relaunch.

## Key Accomplishments

### Autopilot Bug Fixes (9 issues resolved)

1. **Sentinel scoring method mismatch** — `sentinel_questions.yaml` used `substring_match` but `debug_scorer.py` only knows `substring`. All 5 trials scored q=0.0. Fixed all 9 occurrences.

2. **program.md never read by controller** — `CONTROLLER_PROMPT_TEMPLATE` had no `{program}` placeholder. Controller ran without strategic guidance. Added program loading each trial.

3. **TUI log panel empty** — `seed_specialist_routing.py` calls `logging.basicConfig()` at import time (module level, line 64). Python's basicConfig is a no-op when handlers already exist. Autopilot's own basicConfig never took effect. Fixed with `force=True`.

4. **TUI prompt panel shows JSON instead of questions** — No callback from Seeder's per-question loop to TUI. Added `on_question` callback to Seeder class, wired to `tui.set_prompt()`.

5. **TUI prompt panel no autoscroll** — Ported time-based autoscroll from seeding_tui.py.

6. **Journal fields always empty** — `active_flags`, `reasoning`, `hypothesis`, `expected_mechanism` were hardcoded empty. Populated from action dict.

7. **train_routing_classifier.py wrong path** — `structural_lab.py` referenced `orchestration/repl_memory/` instead of `scripts/graph_router/`.

8. **web_research tool not registered** — Defined in `src/tools/web/research.py` but never added to `tool_registry.yaml`. Models hallucinated the tool 144 times in REPL. Added YAML registration.

9. **Standalone TUI support** — Added prompt tap file (`/mnt/raid0/llm/tmp/autopilot_prompt_tap.txt`) so TUI can run independently of autopilot process.

### Program.md Rewrite

Complete strategic rewrite based on user direction:
- **Expanded autonomy**: Autopilot can modify ANY file in orchestrator codebase (`src/**/*.py` included) with git versioning as safety net
- **Git Safety Protocol**: Commit before eval, revert on regression, checkpoint every 10 trials, tag best configs
- **Eval Trust Boundary**: Only immutable files are scoring/eval infrastructure — everything else is fair game
- **Model Registry/Stack guarded**: 5 explicit rules preventing blind exploration of acceleration flags, NUMA, quantization
- **Cost proxy**: Changed from vibes-based multipliers to wall-clock slot occupancy formula
- **Tier reordering**: Feature flags moved from Tier 5 → Tier 2 (hot-swap, zero restart)
- **Known Dead Ends**: All exhausted approaches documented to prevent re-exploration
- **Production Flow**: New section explaining end-to-end optimization target
