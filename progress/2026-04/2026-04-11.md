# Progress — 2026-04-11

## Session 1 — v3 Hybrid SSM+MoE Regression: Root Cause Found & Fixed

### Problem

All hybrid SSM+MoE models (Qwen3.5-35B-A3B, Qwen3-235B-A22B, Qwen3-Next-80B-A3B, Qwen3-VL-30B-A3B) produced empty or gibberish output on the v3 production binary (`production-consolidated-v3`, build 8755, commit `0ddff9ed1`). Pure MoE and dense models worked fine on the same binary. This made the frontdoor (8070) and architect_general (8083) completely non-functional — the stack was unusable for general workloads.

### Root Cause

**Uninitialized `cparams.freeze_recurrent` bool** in `llama_cparams` struct.

The `freeze_recurrent` field was added in commit `02914928b` (SSM state checkpointing for speculative decoding on hybrid models) to gate recurrent state writes:
```cpp
if (!cparams.freeze_recurrent) {
    ggml_build_forward_expand(gf, ggml_cpy(ctx0, new_state, ...));
}
```

But it was never initialized to `false` in the `llama_context` constructor. When heap memory contained a non-zero value, all SSM/recurrent state writes were silently skipped. The model's Delta Net recurrent layers never updated their state, causing garbage output from the very first token.

Only hybrid SSM+MoE models were affected because they are the only architecture with recurrent layers guarded by this flag. Pure MoE and dense models have no recurrent state.

### Investigation Path

1. Confirmed bug on running frontdoor (8070): gibberish reasoning, empty content
2. Confirmed v2 binary (b8219) produces correct output with same model
3. Eliminated KV quantization (`-ctk`/`-ctv` removed) — still broken
4. Eliminated flash attention (disabled) — still broken
5. Eliminated attention rotation (`LLAMA_ATTN_ROT_DISABLE=1`) — still broken
6. Eliminated fused GDN op (`cparams.fused_gdn_ar/ch = false`) — still broken
7. Eliminated recurrent layers entirely (zeroed output) — **still broken from attention layers alone** — this was a red herring; on further investigation the attention layers were fine
8. Binary bisected 563 commits: v2 base → midpoint b8474 (OK) → b8614 (OK) → b8684 (OK) → b8714 (OK) → b8734 (OK) → `0286eaeaa` (OK) → `10f31f752`+ (broken, build failures at intermediate commits)
9. Diffed the 7 final commits — found `freeze_recurrent` field added without initialization
10. One-line fix: `cparams.freeze_recurrent = false;` — verified correct output

### Task 9: LangGraph Phase 3 Flag Flips

**What**: Enabled all 7 per-node `ORCHESTRATOR_LANGGRAPH_*` feature flags in `orchestrator_stack.py`, completing the Phase 3 migration from pydantic_graph to LangGraph backend.

**Bug found & fixed**: `_run_via_langgraph()` in `nodes.py` had an append-field delta bug. LangGraph nodes return deltas for append-reducer fields (role_history, delegation_events, etc.) via `state_update_delta()`, but `lg_to_task_state()` did a full `setattr`, overwriting the original list with just the delta. Fix: save originals before the LG call, then prepend them after `lg_to_task_state` runs.
