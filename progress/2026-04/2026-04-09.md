### Omega Root Cause Audit

Investigated WHY tools hurt. **The only tools being called in REPL mode are `web_search` and `web_research`** — the model defaults to googling answers instead of reasoning:

| Suite | Direct | REPL | Tool | Calls | Diagnosis |
|-------|--------|------|------|-------|-----------|
| coder | 96% | 52% | web_search/research | 2222 | Model googles code instead of solving |
| math | 74% | 49% | web_search/research | 2178 | Model searches math answers instead of computing |
| agentic | 100% | 48% | web_search/research | 405 | Web noise degrades already-correct answers |
| general | 76% | 50% | web_search/research | 2241 | Web results override correct parametric knowledge |

**Root cause**: Not "tools are bad" — the model **over-relies on web search in REPL mode**. Fix: disable web_search for suites where the model has the knowledge, or prompt to prefer reasoning over search.

## Session 3: Web Search Over-Reliance Fix + Arm B Launch

### Problem

Package B Omega audit showed 7/10 suites where REPL mode hurt accuracy. Root cause: model web-searches everything instead of reasoning.

### Investigation

1. Fixed `rules.md` — added tool selection priority hierarchy. **No effect** (100% web search persisted in Arm B preliminary check).
2. Traced prompt loading: `builder.py:_resolve_rules()` falls back to `DEFAULT_ROOT_LM_RULES` in `constants.py` when `config.rules_file` is not set. **The `rules.md` file was never loaded.**
3. Found smoking gun in `constants.py:120`: `"When in doubt, search first"` + `"Use web_research for: any factual question"`.
4. Replaced with priority-ordered guidance: compute > direct answer > reason > search only for genuine gaps.

### WS-3: Cascading Tool Policy for Web Denial (P8b)

Belt-and-suspenders enforcement for the web search over-reliance fix (WS-1). Package B Phase 4 Omega audit found 7/10 suites where REPL mode hurts accuracy because models web-search instead of reasoning.

**Implementation**: All infrastructure already existed (`tool_policy.py`, `ToolRegistry` cascading support, `no_web` context handling). This was a wiring task:

1. Added `NO_WEB_TASK_TYPES` constant to `tool_policy.py` — frozenset of task types where web tools hurt accuracy: `math`, `coder`, `thinking`, `instruction_precision`
2. Added `tool_context` parameter to `REPLEnvironment.__init__`
3. Added `context` parameter to `ToolRegistry.invoke()` and `ToolRegistry.list_tools()`, threaded to `can_use_tool()`
4. Threaded `self.tool_context` through `_invoke_tool` and `_list_tools` in `context.py`
5. All 4 REPL creation sites (`repl_executor.py` ×2, `stream_adapter.py`, `stages.py`) derive `no_web` from `routing.task_ir["task_type"]`
6. Enabled `cascading_tool_policy` feature flag by default (previously validated in prod: PASS -15.3s latency)

**Tests**: 5 new tests in `test_tool_policy.py` — invoke with no_web denies web tools, list_tools with no_web excludes web tools, NO_WEB_TASK_TYPES constant validation. 32 total tests, all passing.

**Next**: WS-2 re-measurement — re-run Omega benchmark to validate combined WS-1 + WS-3 fix. Target: REPL ≥ direct on ≥5/10 suites.
