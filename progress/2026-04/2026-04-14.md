## Session 3 — Wiki Intake URL Linking

### Problem
Wiki articles referenced intake entries by bare ID (e.g., `[intake-191] TurboQuant`) with no clickable link to the actual paper. Humans browsing on GitHub had no way to navigate from a wiki article to the source literature.

### Root Cause
Compilation agents output `[intake-NNN]` as plain text references. The `research/intake_index.yaml` has a `url` field for every entry (346 entries), but the compile workflow did not inject those URLs.

### Changes

| Repo | File | Change |
|------|------|--------|
| epyc-root | `wiki/*.md` (19 files) | Converted 164 `[intake-NNN]` bare references to `[intake-NNN](url)` markdown links using URLs from `intake_index.yaml` |

### Results
- 164 intake references now link directly to paper URLs (arxiv, GitHub, blog posts)
- 19 of 24 wiki articles updated (5 articles had no bare intake refs)
- Example: `[intake-191]` → `[intake-191](https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/)`
