# Progress — 2026-04-16

## Session 1 — Specialist REPL Wasted-Turn Diagnosis and Fix

### Problem
TUI observation: whenever architect_general delegated to coder_escalation, the inference stream appeared to loop back and produce the same answer twice. Investigation of `inference_tap.log` (966K lines, 3227 coder_escalation calls) revealed 3 bugs causing ~810 wasted REPL turns (25% waste rate).

### Root Cause Analysis

| Bug | Occurrences | Root Cause | Wasted Time |
|-----|-------------|------------|-------------|
| `NameError: name 'solution'` | 473 | `extract_code_from_response` drops bare `"""` lines, breaking triple-quoted string assignments. Salvage path extracts only `FINAL(solution)`. | 5-20s each |
| `Unknown tool: run_python_code` | 182 | `CALL("run_python_code")` routes through tool registry where it's not registered. The function exists as a REPL global but `_invoke_tool` only searched the registry. | 5-20s each |
| Dedup guard wastes a turn | 63 | Comment said "break the loop" but code said `continue`, burning another LLM call on identical code. | 5-15s each |

### Changes

| File | Change |
|------|--------|
| `src/prompt_builders/code_utils.py` | Added bare `"""` / `'''` as valid code continuation in `_looks_like_code_line` |
| `src/repl_environment/context.py` | Added `ValueError` fallback in `_invoke_tool` to resolve REPL globals when tool registry lookup fails |
| `src/api/routes/chat_delegation.py` | Changed dedup guard from `continue` to `break`, returning existing output as rescued report |
| `src/api/routes/chat_delegation.py` | Added `repl_turn_errors` list tracking per-turn errors in `_run_specialist_loop`, propagated to `delegation_events` |
| `src/pipeline_monitor/anomaly.py` | New `specialist_repl_errors` anomaly signal (weight 0.5) + `detect_specialist_repl_errors` detector |

## Session 2 — Frontdoor web_search Unavailable

### Problem
Frontdoor model wasted hundreds of tokens reasoning about Polish/German place names (and got the answer wrong) when a single web search would have resolved it instantly.

### Root Cause
Three-way mismatch between prompt, REPL globals, and tool registry:
- `root_lm_system.txt` line 3 says "use web_search()" but the `== FUNCTIONS AVAILABLE ==` section listed no web tools
- REPL globals had `web_fetch` (URL fetch) and `batch_web_search` (batch, feature-gated) — no `web_search`
- Tool registry had `web_search` but only callable via `CALL()` syntax, not as a direct function
- Model saw the mention but no definition → ignored it → hallucinated from parametric knowledge

### Changes

| File | Change |
|------|--------|
| `src/repl_environment/combined_ops.py` | Added `_web_search(query, max_results)` wrapping `tool_registry.invoke("web_search")` |
| `src/repl_environment/environment.py` | Registered `web_search` in REPL globals |
| `src/prompts/root_lm_system.txt` | Added `web_search()` and `web_fetch()` to `FUNCTIONS AVAILABLE` section with examples |

## Session 3 — Specialist Delegation Prompt: Role-Aware Preambles

### Problem
`_build_compact_specialist_prompt` gave every specialist the same preamble: "Execute the delegated coding task quickly." worker_explore got told to write code when it should be searching the web. worker_fast got no tool documentation. Math questions like "3^4 + 4^3" were answered with verbose LaTeX reasoning instead of deterministic Python computation.

### Root Cause
- Hardcoded coding-specific preamble for all roles (chat_delegation.py:253)
- No web tool documentation in specialist prompt despite REPL having `web_search`, `web_fetch`, `web_research` available
- No guidance to use Python REPL for math/computation

### Changes

| File | Change |
|------|--------|
| `src/api/routes/chat_delegation.py` | Added `_CODER_ROLES`, `_SEARCH_ROLES` constants. `_build_compact_specialist_prompt` now selects preamble by role: coders get coding instructions, search/worker roles get web tool documentation + REPL math guidance |
