## Session 1 — Orchestrator Codebase Refactoring Audit

### Problems
1. Subtle diagnostic bugs are hard to identify and surface — the codebase silently degrades quality without any operator-visible indicators
2. 437 `except Exception` handlers across 133 source files catch errors and return defaults instead of surfacing failures
3. Log levels are inverted: routine inference events at WARNING, actual failures at DEBUG
4. Benchmark reward injection is fire-and-forget — MemRL can train on incorrect data without anyone knowing
5. `graph/helpers.py` (2,413 lines, 58 functions) is a god module spanning 7 domains with 34 bare exception handlers
6. Cross-repo file duplication between orchestrator and inference-research has already diverged

### Root Cause
Systemic "defensive programming" pattern: catch Exception broadly, return empty/default values, continue. This creates a cascade where errors propagate upward as silent quality degradation rather than loud failures that draw attention.
