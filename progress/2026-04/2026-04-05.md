intake-262 discovered via expansion from intake-261 (cited as baseline).

### Deep Dive Findings — 4 Transferable Patterns

Deep analysis of Skill0 and AgentOCR extracted four patterns directly applicable to context-folding:

1. **Helpfulness-driven segment scoring** (Skill0): `Δ_k = accuracy_with - accuracy_without` per segment. Their ablation: ranking by Δ_k is THE critical component (-13.7% without it). Transferred to Phase 2c as `segment_helpfulness()` with heuristic scoring (recency, reference overlap, outcome, content sensitivity) + LLM-based calibration for ground truth.

2. **Hash-based segment dedup** (AgentOCR): Content-hash cache achieves 20.79x speedup, 26.82% memory savings. Transferred to Phase 1+ as `SegmentCache` with SHA-256 keys, text normalization (ANSI strip, whitespace collapse), per-session lifecycle.

3. **Free-zone compression threshold** (AgentOCR): c_t ≤ 1.2 is free (95% retention, ~55% savings). Beyond 1.5x, steep nonlinear degradation. Text-sensitive tasks degrade ~3x faster than scene-understanding. Transferred to Phase 2b as a 5-level compression sweep methodology, with expectation that coding tasks have narrower free zone (~20-30%) than exploration (~40-50%).

4. **Role-aware compaction + intermittent reward scheduling** (both papers): AgentOCR finds dense compression reward (K=1) causes collapse (45.3% → 78.2% at K=5). Skill0 uses task-specific budget schedules ([6,3,0] vs [5,3,0]). Transferred to Phase 3b as `CompactionProfile` per role and Phase 3c as `CompactionQualityMonitor` with K-interval scheduling.

## Session 3: LangGraph Migration Phase 1 — Hybrid Bridge

### Implementation

Completed Phase 1 of the LangGraph migration — full hybrid bridge with all 7 nodes replicated as LangGraph node functions, feature-flagged backend selection, and comprehensive test coverage.

| Component | File | Lines | Purpose |
|-----------|------|-------|---------|
| State | `src/graph/langgraph/state.py` | ~200 | `OrchestratorState` TypedDict + 3 custom reducers + converters |
| Nodes | `src/graph/langgraph/nodes.py` | ~570 | 7 async node functions calling `_execute_turn()` |
| Graph | `src/graph/langgraph/graph.py` | ~180 | `StateGraph` builder, `run_task_lg()`, edge validation |
| Bridge | `src/graph/langgraph/bridge.py` | ~50 | `run_task_auto()` — dispatches by feature flag |
| Tests | `tests/unit/test_langgraph_migration.py` | ~280 | 24 tests (state, edges, bridge, reducers, construction, flags) |

### Key Design Decisions

1. **Node functions call same `_execute_turn()`** — zero logic duplication. LangGraph nodes reconstruct a duck-typed `ctx` via `SimpleNamespace` matching `GraphRunContext[TaskState, TaskDeps]`.
2. **`next_node` field for routing** — each node function sets `next_node` in the returned state update. A single conditional edge router (`_route_next`) dispatches all 7 nodes.
3. **Config constants moved to `configurable`** — think_harder_min_* fields (5) passed via LangGraph config, not checkpointed state.
4. **`_result` dict in terminal state** — `_handle_end()` stores TaskResult fields in state for extraction by `run_task_lg()`.
5. **RunnableConfig typing** — node signatures use `RunnableConfig` (not `dict`) to suppress LangGraph warnings.

- Zero regression
