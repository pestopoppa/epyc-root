### LC-5: Fix health probe `full:` prefix URL bug (P9)
- **File**: `epyc-orchestrator/src/api/routes/health.py`
- **Bug**: `_probe_core_backends()` passed raw `full:http://...` URLs to `_probe_backend()`, producing invalid probe URLs like `full:http://localhost:8070/health`
- **Fix**: Strip `full:` prefix and take first URL from comma-separated lists before probing
- **Impact**: Health endpoint no longer reports false "degraded" status when all backends are healthy

### LC-4: Shared-result cache for delegation (P9)
- **New file**: `epyc-orchestrator/src/delegation_cache.py`
- **Design**: In-memory SHA-256 keyed cache (`brief_normalized[:200] + "|" + delegate_to`)
- **Specs**: 1h TTL, 200 max entries, LRU eviction, skip caching error reports
- **Integration**: Wired into `chat_delegation.py` architect loop — cache check before `_run_specialist_loop()`, store after `_compress_report_for_loop()`. On hit, skips full specialist execution.
- **Diagnostics**: `delegation_cache_hits` reported in `delegation_diagnostics` via `delegation_stage.py`
- **Tests**: 10 tests in `test_delegation_cache.py`

### AP-11: Git worktree isolation for PromptForge (P8)
