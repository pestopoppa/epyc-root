## AutoPilot planner: critic-`unavailable` split + tightened uncritiqued dispatch + #776 fix + resume

### Planner safety patch (operator-directed, replaces the earlier "proceed for low/medium risk" idea)
Split a critic **failure** from a genuine **reject**, and tightened dispatch because *risk class alone is not a sufficient guard* (the @708 failure was "low-risk" seed looping):
- `reject` (genuine, parsed): unchanged → safe `seed_batch` fallback, draft not run.
- critic **failure** (timeout/empty/unparseable/circuit-open) on a TRUSTED primary draft → new verdict `"unavailable"`, **keep the draft** (no stale `seed_batch` substitution; that substitution is what re-triggered the @708 `critic_reject_loop`). Because the draft is kept (`action == draft_action`), critic timeouts no longer increment `consecutive_rejected_drafts`.
- `uncritiqued_dispatch_block_reason` (now takes `is_blacklisted` / `is_repeated` from the caller): `unavailable`+HIGH → pause; `unavailable`+`seed_batch`/passive-low-risk → pause unless explicitly observational+one-shot (`OBSERVATIONAL_ACTIONS`, empty); `unavailable`+MEDIUM experiment → proceed ONLY IF novel+non-looping (not blacklisted, not a recurring-invalid signature, carries a `falsifier`); shadow mode never blocks. Case A (`crit is None`, primary failed → fell back to critic) still pauses (option-3).

Codex produced a *valid* fenced `{"type":"train_routing_models","min_memories":250000}`; the validator (`controller_io.py:729`) caps `min_memories` at **100000**. The prompt showed the live corpus as huge (`memory_count=268080`) and the only example as `min_memories: 500`, so codex inferred "use the current corpus" and crossed a hidden cap. Three deterministic prompt-contract fixes:
1. action-example line (`autopilot.py:627`): "min_memories: integer 1-100000; default 500. Do NOT set it to the current memory_count…".

3. `_draft_unusable_reason()`: surface the EXACT validator error (`min_memories must be <= 100000; got 250000`) into `fallback_reason` → log + next-trial "Last Non-Executing Action" feedback, instead of opaque "invalid action".
