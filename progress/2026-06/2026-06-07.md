## Autopilot per-suite regression gate — resolution-aware fix (trial-707 deadlock)

**Trigger:** Autopilot self-halted 2026-06-06 10:36 @ trial 708 on the `critic_reject_loop`
guard (4 consecutive planner drafts rejected). Operator review requested.

**Root cause (confirmed, not hypothesized).** The per-suite regression gate compared
per-suite scores against a fixed `PER_SUITE_REGRESSION = -0.1` floor. On a hybrid T1 eval
(`hybrid_eval(seed=42, t1_n=50)` → ~2 questions/suite over 19 suites) each suite score is
`fraction_correct * 3.0`, quantized to `{0.0, 1.5, 3.0}`. A single correct→incorrect flip is
a **−1.5 swing — 15× the −0.1 floor** — so the gate fired on essentially every seeder trial
(trial 707: hotpotqa/agentic/mode_advantage_hard all "−1.500"). Every such trial was then
`mad_noise`-excluded, never admitted, baseline never moved; the planner kept proposing
`seed_batch`, the critic kept rejecting, and the meta-guard correctly halted. **The planner's
"eval-gate/question-draw artifact" diagnosis was right; the codex critic's "broken-instrument
narrative, no evidence" rejection was wrong** — the evidence was the quantized per-suite METRIC
lines. Overall q=1.674 vs baseline 1.658 (flat-up), reliability 0.977 — no real regression.

**Secondary bug (found during fix).** `classify_learning_exclusion()` tagged a trial `mad_noise`
(benign, Pareto-admissible representative) *before* checking `verdict.passed`. Trial 707 FAILED
three per-suite checks yet was admitted as a "trusted within-noise representative." The MAD test
is quality-only and must not launder a failed verdict onto the frontier.

- `safety_gate.py` — `per_suite_regression_threshold(result_n, baseline_n)` = `−max(0.1, 3/result_n, 3/baseline_n)`
  (coarser single-flip quantum governs; empty counts ⇒ unchanged −0.1 floor). `EvalResult.per_suite_counts`
  field; `Baseline.per_suite_counts_by_tier` tracked + persisted (load/save/state/update_tier) + accessor.
  Gate now logs the resolution-aware threshold and `n_result/n_baseline` (self-diagnosing).
- `learning_exclusions.py` — `mad_noise`/`reproduction_confirmed` benign **only when `verdict.passed`**.
- `autopilot.py` — failed verdict that's not a benign exclusion now SKIPS the clean archive/baseline
  update branch (never raises baseline / admits on a failure); not tagged `bug_corrupted` (real failure ≠ corrupt data).

## Autopilot dashboard plots — un-freeze + always-draw the Pareto frontier (recurring report)

**Trigger (operator, recurring):** GEPA+Pareto dashboard plots "not updated since trial ~700 though 1000
ran", and the Pareto plot shows T0/T1/T2 scatter but **no frontier line**. Reported multiple times — prior
fixes kept hitting the wrong surface (there are TWO: lifecycle-coupled PNGs vs the live SVG `/pareto` panel).

**Root cause (confirmed, not hypothesized).**
1. *PNG freshness is lifecycle-coupled.* `autopilot.py` regenerated plots ONLY at `trial_counter %
   PLOT_INTERVAL == 0`, fired from inside the loop. Any stop on a non-multiple-of-10 (pause, SIGTERM,
   max-trials, internal halt) froze the PNGs at the last decade. Direct evidence: journal/state reached
   707/708 but `orchestration/autopilot_plots/*.png` were stuck at the 06-06 08:21 **trial-700** render.
2. *Static `pareto_frontier_2d.png` had no frontier line* — `progress_plots.plot_pareto_frontier_2d` was
   scatter-only (no `ax.plot`). "Scatter but no frontier" was literally the code.
3. *Live `/pareto` capped at `state_trial_counter`* (`dashboard.py` → `journal_reconstruction.py:110`).
   Benign when state leads the journal (the happy path, save_state every trial), but a crash/rewind between
   saves leaves the counter stale → silently drops every newer journaled row. Latent cause of the live
   freeze; not reproducible against current data (state 708 ≥ journal 707), so hardened, not "fixed-by-guess".

**Changes (epyc-orchestrator, uncommitted on main — coexist with the per-suite-regression work above):**
- `autopilot.py` — `_refresh_plots(sync,reason)` helper + wall-clock trigger (`PLOT_MAX_AGE_S=600`) so a long
  mid-decade gap still refreshes; synchronous final render on **pause-entry, SIGTERM/SIGINT, max-trials,
  shutdown** (`PLOT_SYNC_TIMEOUT_S=180`). Plots can no longer freeze wherever the loop stops.
- `progress_plots.py` — `_frontier_envelope_2d()` + draw the 2D speed×quality envelope as a dashed
  "Pareto frontier" line over the scatter.
- `dashboard.html` `renderParetoScatter` — per-tier frontier line is now the tier's 2D envelope over
  frontier ∪ dominated, so a line shows even when the archived frontier is a single point. Summary surfaces
  stale-state / corruption-exclusion warnings.
- `dashboard.py` `/pareto` — dropped the `max_trial_id=state_trial_counter` cap (journal is source of truth);
  emit `stale_state_warning`, `journal_max_trial_id`, `state_trial_counter`, `exclusions`.
- `journal_reconstruction.py` — exclusion telemetry: `journal_max_trial_id` + `exclusions{bug_corrupted,
  truncated_above_cap, max_trial_id_cap}` so a stale counter / mass `bug_corrupted_by` drop is visible, not silent.
- Tests: `test_progress_plots_filters.py` (+3: envelope logic, degenerate cases, real-render smoke),
