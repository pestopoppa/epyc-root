## W4a baseline promotion ledger events documented

Documentation sidecar for `epyc-orchestrator` commit `0202835` (`Journal
baseline promotion events`). Scope remained root governance documentation only;
no code repo files were edited.

### Landed in `epyc-orchestrator`

- `ExperimentJournal` now has generic `append_ledger_event(...)` and
  `ledger_events(...)` plumbing.
- Typed baseline promotion events are available via
  `BASELINE_PROMOTION_EVENT_TYPE = "baseline_promotion"`,
  `append_baseline_promotion_event(...)`, and
  `baseline_promotion_events()`.
- Baseline-promotion events are ledger rows with no `trial_id`, so they do not
  count as trials, do not advance `next_trial_id`, and do not affect
  supersession folds.
- `autopilot.py` appends a baseline-promotion event only when the existing
  `SafetyGate.update_baseline(...)` result has `updated=True`; acceptance and
  rejection logic are unchanged.
- The event append happens after `archive.save(state)` and `save_state(state)`
  and is wrapped so event-write failures cannot change live decisions.
  `autopilot_state.json:baseline_state` remains authoritative for W4a; events
  are evidence only.

This is a W4a evidence-event scaffold, not baseline-as-fold closure.
