  `fold_supersession_events()` is now used by

  non-trial event rows, and skip GEPA recent trials superseded as

- `d21bbee` (`Fold supersessions in offline analytics`):
  `scripts/autopilot/paired_stats.py`,
  `scripts/autopilot/item_analytics.py`, and
  `scripts/analysis/task_rate_goodput_replay.py` fold supersession events at
  journal read/report boundaries. Paired per-question stats, item analytics,
  and task-rate/goodput replay tables now match the supersession-aware
  archive/dashboard state and ignore non-trial event rows as analysis inputs.
- `965f507` (`Fold supersessions in preflight blacklist audit`):
  `scripts/autopilot/preflight_audit.py::audit_blacklist()` folds supersession
  events before collecting corrupted trial IDs, so blacklist entries sourced
  from trials later superseded as corrupted are caught by preflight.
- `702d785` (`Default journal scrub to append-only events`):
  `scripts/autopilot/scrub_journal.py` now appends supersession events by
  default. Existing trial rows are rewritten only with explicit legacy
  `--rewrite-in-place`; `--append-event` remains as a compatibility no-op.
- `0f07644` (`Filter superseded trials from strategy retrieval`): first W6
  strategy-provenance read guard. `StrategyStore.retrieve()` now accepts
  optional `excluded_trial_ids`, and `_build_mutation_context` derives that set
  from the supersession-aware journal view so PromptForge/CodeMutation
  past-strategy context skips strategies sourced from trials now marked
  `bug_corrupted_by` by append-only supersession events. This does not complete
  W6; provenance IDs and multi-trial evidence filtering remain open.
