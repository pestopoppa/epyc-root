## AutoPilot planner audit → invalid-action feedback loop fixed (control-loop repair)

Audited autopilot + planner logs (`autopilot.log` 8.4 MB, `planner_archive.jsonl` 1801 planner calls / ~11 h, journal 379 trials, state at trial 500). Root cause of the run dead-ending at `max_trials` was **not** planner intelligence — the control loop hid the only actionable error signal, so the planner kept sampling from an impossible action space.

### Findings (evidence-backed)
- **Terminal deadlock:** trials 488–500 (and 119 total across the run) emitted the byte-identical `{structural_experiment, flags:{graph_router:true}}`, rejected **every time** by the flag validator (`graph_router feature requires specialist_routing feature`). The validator error — which states the exact fix — was logged then **discarded** (`actions.py` returned bare `None`; main loop did "increment counter, continue"). ~108 tail trials (393–500) produced **zero** journal rows (counter/journal gap).
- **Blacklist blind spot:** `append_blacklist` only fired on 3 consecutive *safety-gate* failures of *executed* trials; an action that never executes never reached the gate → never blacklisted → infinite repeat possible.
- **Critic was shadow-only:** `AUTOPILOT_PLANNER_MODE=shadow_critique` (default) → `_reconcile` never applied `revised_action`/reject-fallback. 320 reject/revise verdicts, 264 dispatched anyway. The codex critic also never identified the real blocker (missing `specialist_routing` dep) — it critiqued the attribution narrative instead.
- **Action mix:** 78% of 863 dispatches were passive (46% seed_batch + 32% distill_knowledge); genuine experiments <5%.

### Fix shipped — **epyc-orchestrator** `c2033e4` (branch `fix/substring-scorer-digit-separators`), 7 files, scoped pathspec
- **`SkipOutcome(status, reason)`** replaces bare `None` for non-executing actions (invalid flags / AP-9 scope / dirty-tree fence / unknown type / handler no-op). First-class outcomes: **journaled** (`JournalEntry.outcome_status` + `INVALID_ACTION`/`DISPATCH_SKIPPED`), **fingerprinted + counted** (`invalid_signature_counts`), **fed back** to the next planner prompt (live flag state + dependency rules + last-invalid feedback sections), **auto-blacklisted** at 2× for stable `invalid` signatures, and **circuit-broken** on a run (`MAX_CONSECUTIVE_SKIP`, mirrors the meta-action guard).
- **`propose_flag_experiment` now validates the MERGED candidate** (`{**current_flags(), **proposed}`), mirroring POST `/config`, so a dependent flag can be enabled one-at-a-time (the documented two-step was previously impossible — `Features(**flags)` defaulted everything else to False). Unreadable live state → status `error` (non-blacklisting), not `invalid`, so a transient blip can't ban a valid flag. Added `current_flags()`/`flag_schema()`.
- **Experiment-quota ceiling:** once memory ≥ `QUOTA_MEMORY_THRESHOLD` (2000), >`MAX_CONSECUTIVE_PASSIVE` (3) consecutive seed/distill actions are replaced with a self-configuring `numeric_trial` — separates memory maintenance from the optimization budget.
- **Tests:** new coverage for the SkipOutcome contract, merged-validation (single-dependency valid / genuine-violation invalid / unreadable→error / unknown→invalid), error→skipped mapping, quota policy, and feedback renderer. Scoped suites green (48 passed); broad autopilot/journal sweep green (the one `test_features` failure is a pre-existing `ORCHESTRATOR_MOCK_MODE` env-leak, `features.py` untouched).
