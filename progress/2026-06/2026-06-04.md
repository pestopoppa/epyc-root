| # | Snag | Resolution |
|---|------|------------|
| 1 | Host pkg is **earlyoom v1.7** — lacks `--ignore` / `--sort-by-rss`; its oom_score/badness mode kills firefox renderers (self-elevated badness 822 for a 77 MB tab) before the real memory hog | **Built ≥1.8 from source** (zero-dep C, `make && sudo make install` → `/usr/local/bin/earlyoom`, unit at `/etc/systemd/system/earlyoom.service`). | 2 | Service failed `217/USER` ("Failed to determine user credentials"); `sudo -u node` → "unknown user node" | **The host has no `node` user** — `node` is the *devcontainer's* name for uid 1000; on the host uid 1000 = **`daniele`**. | 3 | Packaged unit is `DynamicUser=true` + `ProtectSystem=strict` → `-N` hook (random UID, read-only FS) can't write the node-owned audit log | Drop-in override: `DynamicUser=false`, `User=daniele`, `ReadWritePaths=…/logs`, `Nice=-20`, `OOMScoreAdjust=-100`. |
| 4 | v1.7 default `-s 10` + 8 GB lightly-used swap would make earlyoom **never fire** on memory pressure | `-s 100,100` (percent) neutralizes the swap gate so memory gates alone. |

**Final armed config**: `-M 41943040,20971520 -s 100,100 -r 60 --sort-by-rss --ignore '^(llama-server|sd-server)$' --prefer '^llama-bench$' -N …/earlyoom_audit.sh`. ### Durable control-plane protection — `epyc-orchestrator` (commit `2a40260`)

The one-shot `choom -1000` doesn't survive an API restart, and the API procs are `comm=python` (can't be earlyoom `--ignore`'d by name — they collide with runaway python evals). Wired `oom_score_adj=-1000` into the launcher:

| Repo | File | Change |
|------|------|--------|
| `epyc-orchestrator` | `scripts/server/stack_processes.py` | New `set_oom_score_adj(pids, adj=-1000)` — best-effort `sudo -n choom` (falls back to `sudo -n tee /proc/<pid>/oom_score_adj`); lowering below 0 needs `CAP_SYS_RESOURCE`, so it uses the same NOPASSWD pattern as `stack_host.py`/`host_health.py`. Missing/denied sudo logs + skips (never fails a stack start). |

| `epyc-orchestrator` | `tests/unit/test_stack_processes.py` | 5 unit tests (choom path, tee fallback, no-sudo no-op, best-effort-on-failure, empty). |
