## AutoPilot dashboard GEPA/Pareto plot panels silently frozen since 2026-05-31 — missing matplotlib dep

**Symptom (operator report)**: dashboard on trial 481 but the GEPA+Pareto plot panels hadn't gained data since ~trial 363.

**Investigation discipline**: a first Explore pass proposed a "`continue` bypasses the `%10` plot check" code-flow theory. Rejected on primitive evidence — it would require all 12 multiples of 10 between 360→481 to coincide with a skipped trial. The journal was being written *today* (`trial_id 392 @ 10:13:34`) while every PNG was frozen at `2026-05-31 12:48:25`, so the trigger was firing and the *render* was dying. Ran the exact spawned command to get the real error.

### Root cause
The 6 dashboard PNGs are rendered by a fire-and-forget `autopilot.py plot` subprocess every `PLOT_INTERVAL=10` trials (`autopilot.py:2173`). `matplotlib` was **used by `progress_plots.py` but never declared** in `pyproject.toml`. A venv rebuild on 2026-05-31 12:07 (`uv.lock`/`pyproject` mtimes, ~40 min before the freeze) dropped it. Every render then raised `ModuleNotFoundError`, which `generate_all_plots()` caught and **still exited 0**. The async reaper (`phase_status.reap`) only warns on non-zero exit → logged `[async] plots-trial-N complete`. Completely silent; panels froze at the last good render (~trial 360) while the journal kept advancing. The "363 vs 481" gap was two healthy counters: live `trial_counter` (state, =489→500) vs journal `trial_id` (metric-bearing only, max 392).

### Changes
| Repo | File | Change |
|------|------|--------|
| `epyc-orchestrator` | `pyproject.toml` + `uv.lock` | Declare `matplotlib>=3.10.9` (`uv add`) so `uv sync` can't silently drop it again. |
| `epyc-orchestrator` | `scripts/autopilot/progress_plots.py` | `generate_all_plots(..., raise_on_error=False)` — re-raises when set; default best-effort behavior unchanged for in-process callers. |
| `epyc-orchestrator` | `scripts/autopilot/autopilot.py` | `cmd_plot` calls with `raise_on_error=True` and `sys.exit(1)` on failure → reaper now logs `[async] plots-trial-N failed rc=1` instead of false "complete". |

### Results
- `matplotlib==3.10.9` installed into the autopilot `.venv`; import confirmed.
- Re-rendered immediately — all 6 PNGs refreshed (data through journal `trial_id 392`).
- Hardening verified: success path exits 0; simulated missing-matplotlib exits 1 (both inner + outer error logs fire); both files `py_compile`.
- **Live proof**: the running autopilot's own cadence render fired at `~12:50:00` (post-install) and **succeeded** on its own — production path works end-to-end, not just the manual run.
- The running autopilot (PID `2492317`, started Jun 4 03:16) then **completed at `trial_counter=500` (its `--max-trials 500`) and exited** — a clean completion, not a halt. No autopilot process is running now.
