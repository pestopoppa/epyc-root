## Problem Statement

Models had **zero routing agency**. All escalation/delegation was hardcoded Python control flow:

1. **`escalate()` was dead code** — set `artifacts["_escalation_requested"]` but `chat.py` never checked it
2. **`recall()` was wasteful** — created fresh `EpisodicStore` + `TaskEmbedder` per call, returned no Q-values
3. **No model self-awareness** — models didn't know their role, tier, or capabilities
4. **No MemRL visibility** — rich Q-value infrastructure (episodic memory, failure graphs, hybrid router) existed but was invisible to models

## Solution

Exposed MemRL intelligence to the REPL environment via 5 model-facing functions, wired artifact checking into chat.py, and injected routing context on turn 0.

### Data Flow (Before → After)

```
BEFORE:
  Request → HybridRouter (pre-model) → Model executes → Error? → EscalationPolicy → retry/escalate
  Model has: recall() [broken], escalate() [dead], llm_call() [blind]

AFTER:
  Request → HybridRouter (pre-model) → Inject routing context → Model executes
  Model can: my_role() → route_advice(task) → delegate(prompt, role) or escalate(reason, target)
  After execute: chat.py checks artifacts → honors model's routing request
  MemRL logs delegation outcomes → Q-values improve future routing
```

## Design Decisions

1. **Streaming parity now**: Both `/chat` and `/chat/stream` have identical routing logic
2. **Tier C guard in `_delegate()`**: Workers cannot delegate. They can use deterministic tools.
3. **No cold-start bootstrapping**: MemRL is pre-seeded (2,714 memories, 13 failure modes). Learns continuously via TD-learning.
4. **Routing context capped at 300 chars**: ~75 tokens overhead on turn 0, negligible impact
5. **Legacy `recall()` fallback**: When retriever is None, creates fresh instances (backward compat)

## Kimi K2.5 Agent Swarm Research & PARL-Inspired Handoff

Researched Moonshot AI's Kimi K2.5 Agent Swarm (PARL — Parallel-Agent Reinforcement Learning) and compared it to our orchestrator. Key findings:

- **K2.5**: Single 1T MoE model (32B active, 384 experts) plays all specialist roles via different system prompts. Up to 100 subagents, 1,500 tool calls per task.
- **Our system**: Many specialized models (0.5B-480B) with hardware-specific acceleration. 2-4 worker parallelism, deterministic quality gates.
- **Key insight**: K2.5's "dynamic agent instantiation" translates to **dynamic prompt specialization** (personas) in our architecture, not dynamic model selection.

### Actionable Learnings Identified (5 Phases)

| Phase | Learning | Value |
|-------|----------|-------|
| 1 | Parallel TaskIR execution (dual paths: ParallelStepExecutor + REPL `llm_batch()`) | HIGH |
| 2 | Critical path metric + `llm_batch()` instrumentation | HIGH |
| 3 | Persona registry (10 personas) + hybrid MemRL auto-selection (Q > 0.6) | HIGH |
| 4 | Staged reward shaping for MemRL (per-session annealing) | MEDIUM |
| 5 | Parallel gate execution (capped at 3, lightweight gates only) | MEDIUM |

### Phase-by-Phase Review Decisions

- **Phase 1**: Both structured (ParallelStepExecutor with wave-based DAG execution) and freeform (REPL `llm_batch()`) paths
- **Phase 2**: Instrument both `src/api.py` Root LM loop (Path A) and `llm_batch()` in `llm_primitives.py` (Path B)
- **Phase 3**: 10 personas (security_auditor, technical_writer, code_reviewer, data_analyst, inference_specialist, benchmark_analyst, computational_physicist, ai_engineer, system_architect, devops_engineer). Hybrid selection: explicit if specified, MemRL auto-suggest otherwise, fallback to vanilla
- **Phase 4**: Per-session annealing reset (lambda 0.3→0.0 over 50 steps, resets each session)
- **Phase 5**: No changes — hardware guardrails already adequate

## Orchestrator Quality Regression Fix — Direct-Answer Mode + VL Pipeline Rewrite

### Problem

Run 20260129_101033 scored **30/82 (37%)**. Two critical findings:

1. **REPL wrapper destroys instruction precision**: Models individually score 10-11/11 on agentic + instruction_precision suites, but only 2-6/11 through the REPL wrapper. The wrapper forces the model to "write Python code → call FINAL(answer)" which adds fatal indirection for simple Q&A tasks.

2. **VL pipeline completely broken**: 6/12 VL prompts returned empty. Root causes: `worker_vision` mapped to port 8082 (Qwen2.5-7B text-only model) instead of port 8086 (Qwen2.5-VL-7B), response key mismatch in handler, and subprocess-based pipeline silently failing.

### Fixes Implemented (3)

| Fix | File | Description |
|-----|------|-------------|
| A: Direct-answer mode | `src/api/routes/chat.py` | `_should_use_direct_mode()` auto-detects when REPL is unnecessary, calls model directly with just the user prompt. Bypasses Python-code wrapper for short prompts without file/tool operation keywords and context < 20K chars. Includes MemRL quality review gate. |
| B: Port mapping | `src/llm_primitives.py` | `worker_vision` → port 8086 (was 8082), added `vision_escalation` → port 8087 |
| C+D: VL pipeline rewrite | `src/api/routes/chat.py` | Rewrote `_handle_vision_request()` to call VL servers directly via `/v1/chat/completions` with multimodal payload (base64 data URI). Fallback chain: 8086 → 8087 → legacy pipeline. Proper MIME detection and error logging. |

### Investigation Details

**Fix A — Why REPL wrapper kills quality:**
- System prompt adds ~900 tokens of tools/rules overhead
- For "What is 2+2? Reply with only the number", the model must generate Python code that produces `"4"` then calls `FINAL("4")` — most models output extra text, tool calls, or JSON metadata
- Same model scores 11/11 on instruction_precision without REPL, 2/11 through REPL
- `_should_use_direct_mode()` returns True for short prompts without file/tool operation keywords

**Fix B+C+D — VL pipeline root causes:**
- Port mapping: `llm_primitives.py` had `"worker_vision": "http://localhost:8082"` — Qwen2.5-7B-Instruct (text-only explore model), NOT the VL model
- Handler: `_handle_vision_request()` called legacy subprocess pipeline (`/vision/analyze` → 404, route was at `/v1/vision/analyze`), then parsed response with wrong key (`data.get("vl", {})` vs `data.get("description")`)
- New handler sends image as base64 data URI in multimodal chat completions format directly to VL servers

### Three-Way Mode Selection

```
Request → _select_mode(prompt, context, state) → "direct" | "react" | "repl"

MemRL path: state.hybrid_router.route_with_mode(task_ir) → (roles, strategy, mode)
Heuristic fallback: _should_use_direct_mode() → _should_use_react_mode() → repl
```

Mode selection criteria:
- **direct**: Short prompts, no tools needed, instruction-following tasks
- **react**: Tool-needing keywords (search, calculate, date) without large context
- **repl**: Large context (>20K chars), file operations, code execution
