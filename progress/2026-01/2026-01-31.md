## Session 1: Phase 3 Validation Overhead Investigation

### Problem

Seeding script (`seed_specialist_routing.py`) showed ~68s overhead on `direct`/`repl` modes but only ~6s on `react` mode. Investigation needed to determine if Python code or server-side.

### Root Cause

**Server-side llama-server overhead, NOT Python code.** The `prompt_ms` and `predicted_ms` from llama.cpp only cover actual model inference. The ~68s is invisible server-side overhead:
- KV cache eviction/reallocation (when `cache_prompt: True`)
- Memory operations (mmap page management, NUMA migration)
- Post-inference cache persistence

### Evidence Pattern

| Previous generation | Overhead | Note |
|---------------------|----------|------|
| 2 tokens | ~6s | LOW |
| 72+ tokens | ~68s | HIGH |
| Cross-server (port 8081 after 8080) | ~68s | **Machine-wide**, not per-server |

### Code Paths Eliminated

All Python code paths investigated and eliminated as overhead source:
- `_architect_plan_review` (not enabled), `_architect_verdict` (skipped for short answers)
- `_formalize_output` (not enabled), `_detect_output_quality_issue` (not enabled)
- `ensure_memrl_initialized` (one-time), `score_completed_task` (lightweight DB)

## Session 4: Health Check Fix + Stratified Sampling + MCP + Requirements Audit

### Problem 1: Cascading Health Check Failure

Seeding script's health check used a single `break` when the orchestrator was unhealthy. After frontdoor:react generated 953 tokens (large KV cache), the orchestrator became briefly unresponsive. The `break` at line 528 exited the entire combo loop — coder_escalation (port 8081), architect_general (port 8083), and architect_coding (port 8084) were ALL skipped despite being on separate servers.

### Fix: Per-Role Skip with Retry

Replaced aggressive `break` with per-role `failed_roles` tracking:

| Change | Detail |
|--------|--------|
| `failed_roles: set[str]` tracking | Skip only remaining combos for the failed role, not all combos |
| 5-second retry with 10s timeout | Brief server unresponsiveness gets a second chance |
| `continue` instead of `break` | Other roles on different servers proceed normally |
