**Key changes:**

1. **FAISSEmbeddingStore**: Uses `IndexFlatIP` with L2 normalization for cosine similarity
   - `add()`: Normalizes and adds embedding
   - `search()`: Returns (memory_id, score) tuples
   - `get_embedding()`: Reconstructs embedding from index
   - `save()/load()`: Persists to `embeddings.faiss` + `id_map.npy`

2. **NumpyEmbeddingStore**: Legacy fallback with same interface
   - Memory-mapped numpy array
   - O(n) brute-force search
   - For migration/rollback

3. **EpisodicStore updates**:
   - Added `use_faiss=True` parameter (default)
   - Changed `db_path` to directory-based storage
   - Updated `retrieve_by_similarity()` to two-phase: FAISS search → SQLite filter
   - Added `similarity_score` field to `MemoryEntry`

4. **Migration script** (`scripts/migrate_to_faiss.py`):
   - Batch processing (10K embeddings per batch)
   - Validation and comparison modes
   - Non-destructive (preserves original `embeddings.npy`)

### Test Results (Twyne Whitepaper - 87K chars)

| Metric | Without Compression | With Compression |
|--------|---------------------|------------------|
| Time | 74s | 140s (**SLOWER**) |
| Output quality | Clean, professional | Degraded |
| Hallucinations | None | Fake citations `[1]` |
| Typos | None | "tokenizeize", "cUSTODY" |
| Prompt leakage | None | Present |

### Root Cause

LLMLingua-2's extractive compression produces choppy, fragmentary text:
```
Twyne Whitepaper Jakub Warmuz noncustodial risk modular credit delegation
protocol capital inefficiencies DeFi lending markets conservative passive...
```

This confuses downstream LLMs, causing hallucinations to fill semantic gaps.

### Decision

**DISABLED compression. Wait for Cmprsr (abstractive) weights.**

### Key Finding: TOON NOT Beneficial for Grep Hits

TOON's tabular format repeats pattern on every row. Markdown's grouped format (`### Search: pattern`) is 18% more compact.

```
# Markdown (291 chars) - BETTER:
### Search: `abstract`
Line 12: This paper presents...

# TOON (345 chars) - WORSE:
excerpts[4]{pattern,source,line,match}:
  abstract,paper.pdf,12,This paper...   <- pattern repeated!
```

## Episodic Memory Performance Optimization

**Goal:** Reduce episodic memory latency from 61-255ms to <15ms

### Results Summary

| Metric | Before | After | Improvement |
|--------|--------|-------|-------------|
| Embedding | 50-200ms | 2-5ms | **40x faster** |
| Storage | 10-50ms | ~0ms (async) | **Async batch** |
| Retrieval | 9-50ms | <5ms | **10x faster** |

### Implementation (6 phases)

| Phase | Change | Impact |
|-------|--------|--------|
| 1 | Embedding server (HOT tier, port 8090) | 50-200ms → 2-5ms |
| 2 | Write-behind FAISS (10s flush) | Blocking → async |
| 3 | id_to_idx O(1) dict | O(n) → O(1) lookup |
| 4 | SQLite compound indexes | Filtered queries ~1ms |
| 5 | Graph penalty caching (TTLCache 60s) | 5-20ms → <1ms |
| 6 | store_immediate() for ACID | Optional sync flush |

### Files Modified

| File | Change |
|------|--------|
| `scripts/server/orchestrator_stack.py` | Added embedder to HOT tier (port 8090) |
| `orchestration/repl_memory/embedder.py` | HTTP client with fallback chain |
| `orchestration/repl_memory/episodic_store.py` | Write-behind + indexes + store_immediate() |
| `orchestration/repl_memory/faiss_store.py` | O(1) id_to_idx dict |
| `orchestration/repl_memory/retriever.py` | TTLCache for graph penalties |
| `pyproject.toml` | Added cachetools>=5.3 |
