## REPL File Access Fix

**Status:** Complete

---

### Task: Fix REPL File Access Failure

**Issue:** REPL mode (`real_mode=true`) failed when attempting to use `peek()` and `grep()` to access files. The orchestrator escalated through multiple tiers but ultimately failed with "No escalation possible from architect_general".

**Root Cause:** The `peek()` and `grep()` functions only operated on the in-memory `context` variable - they had no way to read files from the filesystem. This is the core issue identified in yesterday's log (2026-01-23).

---

### Solution: Add File Access to peek() and grep()

Modified both functions to accept an optional `file_path` parameter:

```python
# Original behavior (still works)
peek(100)              # Read from context
grep("pattern")        # Search context

# New file access
peek(100, file_path="/mnt/raid0/llm/tmp/file.txt")
grep("pattern", file_path="/mnt/raid0/llm/tmp/file.txt")
```

## LightOnOCR 3.4x Slowdown Fix

**Status:** Complete

---

### Task: Fix LightOnOCR Slowdown Through Orchestrator

**Issue:** LightOnOCR-2 was running 3.4x slower through the orchestrator TaskIR preprocessing pipeline compared to direct benchmarks.

| Metric | Direct Benchmark | Via Orchestrator (Before) | Via Orchestrator (After) |
|--------|------------------|---------------------------|--------------------------|
| Speed | 0.17 pg/s | 0.05 pg/s | **0.117 pg/s** |
| 32-page PDF | ~188s | 638s | **276.6s** |

---

### Root Causes

1. **Client-side per-page HTTP requests:** `process_document()` used `ocr_pdf_with_partial_success()` which makes individual HTTP requests per page. The server's `/v1/document/pdf` endpoint handles all pages internally with parallel workers.

2. **Per-page timeout too short:** Server timeout was 120s, but complex pages take 24-30s. With parallel processing, queue delays caused timeouts.

---

### Fixes Applied

| Fix | File | Change |
|-----|------|--------|
| Use server-side processing | `src/services/document_client.py:360` | `ocr_pdf()` instead of `ocr_pdf_with_partial_success()` |
| Increase timeout | `src/services/lightonocr_llama_server.py:53` | 120s → 300s |
| Update registry | `orchestration/model_registry.yaml` | Updated timeout and added quirks |

---

### Infrastructure Decision: No K8s/Terraform

**Evaluated:** Kubernetes + Terraform for service orchestration

3. mmap model loading - HOT/WARM/COLD pools need direct memory access
