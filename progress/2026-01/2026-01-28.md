## Session 9: Implement Prompt Lookup in llama-server

**Objective:** Port n-gram prompt lookup from llama-cli to llama-server, enabling 12.7x summarization speedup in server/API mode.

**Branch:** `feature/server-prompt-lookup` in `llama.cpp-experimental`

### Design

- Prompt lookup AND speculative decoding coexist per-slot
- Try lookup first (free CPU hash op), fall back to draft model if insufficient matches
- Ngram cache built from prompt tokens, updated during generation, cleared on slot release
- Per-request `"lookup": true` toggle; `--lookup` CLI flag sets default
- Reuses existing `i_batch_dft`/`drafted`/`common_sampler_sample_and_accept_n()` verification pipeline (draft-source agnostic)
