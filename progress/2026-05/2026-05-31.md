### Root cause
The safety gate ran with an in-memory `baseline.quality = 9.900` (and `per_suite_quality['coder']

## CORRECTION + real root cause: baseline save-path pollution (gate-lock)

Earlier sections in this file hypothesized the 9.900 baseline came from "stale process
memory." That was incomplete. The decisive live trigger of the gate-lock loop was found
during wrap-up:

**`Baseline.save()` defaulted to `DEFAULT_BASELINE_PATH` and `Baseline` did not remember
the path it was loaded from.** So `SafetyGate(baseline_path=<tmp>)` loaded from tmp, but
`update_baseline()` → `self.baseline.save()` wrote to the REAL
`orchestration/autopilot_baseline.yaml`. Running `test_safety_gate_baseline_eligibility.py`
(fixture quality=2.9, speed=99.0, reliability=0.99) therefore overwrote the production
baseline with **quality 2.9** — above the 2.400 Pareto archive max. The regression gate then
force-reverted every honest trial (2.4 < 2.9 → "regression"), and the planner fell back to
no-op `distill_knowledge`: **77 of 81 trials (190→271) ran zero inference.** Reproduced live
during wrap-up — a pytest run rewrote baseline.yaml to 2.9 twice.

action: no eval, no system change). The trial counter + ~57 orphan git tags (autopilot/trial-215..274)
