## Dirty-tree fence on file-mutating actions (actions.py — commit e58a79c)

Sixth launch blocker. With PromptForge `auto_commit=not dry_run` (True in live
controller runs), a code/prompt mutation executed while its commit target
already has uncommitted changes would sweep that unrelated work into an
`autopilot:` commit. Concrete hit: `src/api/routes/chat.py` is both dirty
(parallel-agent hunk) and in `CODE_MUTATION_ALLOWLIST`.

- `dispatch_action` now calls `_mutation_dirty_target_reason` right after scope
  validation; a dirty commit target → skip the trial (return `None`), never
  mutate. Fires regardless of `auto_commit` and fails CLOSED (any git error →
  treat as dirty → skip).
- Scope matches how each path actually commits:
  - `code_mutation` → single target file (`git add <file>`)
  - `prompt_mutation` / `gepa_optimize` → whole prompts dir (`git add <prompts_dir>`)
  - `structural_prune` → single prompt file (direct write, no forge commit)
- Bug caught by the new tests during review: `actions.py` never imported
  `subprocess`, so the helper would have NameError → fail-closed → silently
  block *all* mutations. Added the import; 18 fence tests + full autopilot suite
  (145) pass.
