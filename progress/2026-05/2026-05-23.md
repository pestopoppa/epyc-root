### Commit `ab889b1` — three small follow-ups

#### Follow-up 1 — inference_tap shim race

**Prior pattern** (commit history pre-dates this session):
```python
# src/inference_tap.py
import importlib as _il, sys as _sys
_real = _il.import_module("src.runtime.inference_tap")
_real.__name__ = __name__
_sys.modules[__name__] = _real
```

Race window between `import_module` returning and `sys.modules` swap. A second thread doing `from src.inference_tap import is_active` could see the half-built shim namespace (no `is_active` binding) and fail with `ImportError: cannot import name 'is_active' from 'src.inference_tap'`. Observed ~1-in-3 during concurrent stress test in this session.

**Fix**:
```python
from src.runtime.inference_tap import (  # noqa: F401  (re-export)
    is_active, stream_mode, should_stream_role, tap_section,
)
# sys.modules swap retained as best-effort; no longer load-bearing.
```

Public symbols eagerly bound at import time on the shim module. Attribute access succeeds regardless of swap state. **20-thread concurrent-import smoke test: 0/20 errors** (was ~1-in-3 before).

#### Follow-up 3 — stale planner session_id auto-clear

Autopilot was hitting the planner with a stale `--resume <session_id>` every trial:

```
ERROR: Controller failed (rc=1): No conversation found with session ID:
       abee6b65-6537-4bc8-b0fa-4456bb2d3551
```

The old session was pruned (the underlying claude CLI doesn't keep arbitrarily-old sessions on disk). The `session_id` was persisted in `autopilot_state.json` and re-sent on every controller invocation, forcing the "No action proposed, defaulting to seed_batch" fallback every cycle. Planner reasoning was effectively dead until manual state reset.

**Fix in `controller_io.py:invoke_controller`**:
```python
if proc.returncode != 0:
    stderr = proc.stderr.read() or ""
    if "no conversation found" in stderr.lower() or (
        "session id" in stderr.lower() and "not found" in stderr.lower()
    ):
        log.warning("Clearing stale planner session_id=%s ...", session_id[:12])
        return "", None    # ← was: return "", session_id
    return "", session_id
```

When the caller's pattern is `state["session_id"] = session_id`, returning `None` clears the stale ref on the next save. Next trial starts a fresh `claude -p` conversation (no `--resume`).

**Verified in live autopilot.log** (this session):
```

WARNING: Clearing stale planner session_id=abee6b65-653 (CLI reports it no longer exists);
         next trial will start a fresh conversation
```

## Session 2 — AutoPilot constrained-creativity planner upgrade

User-driven refactor of how the autopilot's planner samples its next action.
The prior mechanism (`tail_action_candidates` returning 3 random under-used
action types per trial as candidates the controller had to defend) conflated
"unlikely" with "creative" — the LLM was being asked to argue for noise on
equal footing with the obvious answer, with no rubric to judge them.

### Problem framing

User's critique (paraphrased): the far tail of action-space is dominated by
incoherence, not insight. What makes a thought creative is not maximum
distance from the mean but maximum distance subject to truth-preserving
constraints. The deployed mechanism had three structural issues:

1. Tail candidates competed on equal footing with the modal action with no
   rubric to discriminate them → the "Adjacent Possible" enumeration was
   often performative reasoning that picked the obvious action anyway.
2. The mechanism ran every trial including mid-exploit on a working lead →
   wasted prompt budget when not stuck.
3. No falsifier was recorded with the chosen action → next trial's planner
   couldn't see what-was-supposed-to-happen-but-didn't, so the entropy claim
   never became operational.

### Approach

Plan written to `~/.claude/plans/if-you-want-i-lucky-gem.md`, then
implemented in five tracked tasks:

1. Add `falsifier` + `rubric_scores` fields to `JournalEntry` with `.get(field, default)` migration; new `unfalsified_hypotheses()` helper.
2. New `extract_rationale(text) -> dict` parser in `controller_io.py` for the `json:autopilot_rationale` sidecar block.
3. Rework `autopilot.py` prompt + stagnation gate: new constants, conditional lean vs. rich exploration fragment, `_build_exploration_block()` helper, rationale capture wired into trial record.
4. Update `program.md` (no scrub_journal edit needed — `asdict(entry)` handles new fields automatically).
5. Tests — extend `test_autopilot_controller_io.py` (+6) and new `test_autopilot_creativity.py` (10).

### Stagnation gate design

Rich creative protocol activates when any of:

- `hv_slope_10 < STAGNATION_HV_EPS` (Pareto frontier not advancing)
- `journal_trustworthiness["low_signal"]` true (trustworthy trial count < 5)
- Last 3 trials share the same `action_type` (exploit lock-in)

Otherwise the planner sees the lean fragment (single-paragraph "enumerate
3–5 alternatives with one-line reject/accept reasons"). All signals reuse
existing infrastructure (`pareto_archive.geometry()`, `journal.trustworthiness_score()`, `journal.recent(N)`).

### Rich fragment shape

5 candidates under truth-preserving constraints, scored 1–5 on three
**orthogonal** axes (collapsed from the user's draft 6-axis rubric to
prevent score mushing): `info_gain` (novelty + falsifiability +
compression), `coherence` (consistency with last 30 trials + signatures +
blacklist), `usefulness` (Pareto improvement per compute minus
aesthetic-nonsense risk). Tail-sampled under-used action types are passed
as inspiration ("directions explored ≤1 time in last 30 trials"), not as
candidates the model must defend. Fusion preference: if top-2 candidates
can be encoded as one action dominating both, prefer the fusion.
Quote-don't-regenerate rule: chosen action's rubric scores in the
rationale sidecar must be copied verbatim from the candidate table —
anti-drift guard against performative reasoning.

### Falsifier loop

Asking every chosen action to ship with a one-line falsifier creates a
cross-trial feedback loop: trial N commits to a prediction whose absence
would invalidate it; trial N+k's planner sees that falsifier (via the new
`unfalsified_hypotheses()` helper) and grades new candidates against
still-open claims. This is what makes the "real insight lowers entropy
out-of-sample" claim something the system actually checks over time,
rather than a thing we assert.
