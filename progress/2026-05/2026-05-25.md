### Detailed code changes (`3ba5ca1`)

**A. `/dashboard/api/autopilot_progress` rewrite**
- New `_parse_journal_ts()` helper: ISO-8601 strings → unix seconds.
- New `_tail_deep_eval_progress()` helper: tail-reads the last 64 KB of the autopilot stdout log, finds the most-recent `T2 progress: X/Y` marker → authoritative percent for tier-2 evals.
- Per-entry durations now derived as `timestamp[i] − timestamp[i-1]` (autopilot is serial, so gap ≈ runtime).
- Median stratified by `action_type` so a `deep_eval` doesn't get compared against a `numeric_trial` p50.
- Falls back through 3 sources: `log_tail` (exact for deep_eval) → `action_p50` → `aggregate_p50` → `fallback` (hardcoded 1200s). New `percent_source` field tells the operator which one fired.

**B. `dashboard.html` task detail defensive guards**
- `(Array.isArray(d.events) ? d.events : [])` at all 3 reference sites.
- Show "task fetch failed: HTTP \<code>" message when fetch returns non-2xx.
- Distinguish "no events, no slot, no tap match — task may have been pruned" from "task_started.objective (no slot or tap match)".

**C. Routing decisions panel — widen + per-source-aware**
- `_scan_recent_decisions` max_items: 50 → 200 (per the comment at the wrapper, last-50 was overwhelmingly frontdoor).
- New server-side aggregates carried under underscore-prefixed keys in `source_counts_rolling`:
  - `_today_roles`: `{base_role: count}` across the FULL day (not the window) → "role chosen — today (N)" section.
  - `_classifier_conf`: `{high, medium, low}` histogram across classifier-source decisions only (was lumping all sources together → produced misleading "100% no-confidence" rows for learned-source-dominated windows).
- Per-source-aware denominators in the JS: classifier confidence + verifier verdict use `bySource.classifier`, not total.
- Explanatory tooltips: hover over "classifier confidence" → "histogram across classifier-source decisions only (N of M); learned/rules/forced sources do not produce a classifier_confidence value".

**D. Region locks panel — enumerate quartered roles + instance pills**
- `/dashboard/api/region_locks` now also enumerates from `get_instance_regions()` (so quartered roles with no lock files still show, with `wired: false`).
- Builds `pid_to_instance` map by scanning `/proc` for `--port N` → resolving via `_PORT_HINTS` + the `<role>.qN` convention.
- Each lock entry carries `holder_instance_idxs: [int]` resolving holder PIDs back to NUMA_CONFIG instance indices.
- Each role bucket carries `instances: [{idx, regions[], span, is_full}, ...]` + `active_instance_idxs: [int]` for the front-end.
- Front-end renders a strip of instance pills sized by region span (`span=4` ≈ 88px, `span=2` ≈ 44px, `span=1` ≈ 22px). Active pills tinted red. Cell tooltips name the holding instance (e.g. "held by instance 0 · pids 73661" + label = "full"/"half"/"q0").
- Wired roles sort first; unwired roles render with greyed-out cells + `not wired` warn-color badge.

**E. React legacy purge**
- 5 seed entries in `seed_loader.py:122-130` rewritten from `"action": "frontdoor:react"` to `"action": "frontdoor:repl"`, mode field too.
- Defensive `:react → :repl` normalization in `chat_pipeline/routing_decision.py` at the `last_decision_meta` merge point — single chokepoint vs. patching 7 inline write sites in `hybrid_router.py`.
- Static `_normalize_action` classmethod added to `hybrid_router` + applied in `_record_decision_meta` for the primary write path.
