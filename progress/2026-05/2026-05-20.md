## Phase 2 — Research-intake on intake-569 + deep-dive

### intake-569 details

**arxiv:2605.15514** — "RoPE Distinguishes Neither Positions Nor Tokens in Long Contexts, Provably" (Du, Harris, Tian, Huerta, Ronanki, Rongali, Galstyan, Peng — submitted NeurIPS 2026, posted 2026-05-15, **5 days old at intake time**).

**Headline result**: closed-form proofs of 4 RoPE failure modes (Position Inversion, Position Aliasing, Token Inversion, Token Aliasing) plus an **irreducible trade-off** — raising the RoPE base `B` (the move every long-context extension uses) helps token-distinguishing (F3/F4) at the *direct cost* of position-distinguishing (F1/F2). No `B` preserves both.

**Empirical**: all tested 7B–405B models (Llama 3.1, Mistral, Qwen3, DeepSeek-v3, Kimi K2.5, GPT-OSS) collapse to ~0.25 chance accuracy on a 4-element position-indexing task by **4K–8K tokens** — far below their nominal 32K–128K context windows.

### Du–Wang triangulation (key insight)

Two independent papers, different methodologies, converge on the same regime boundary:
- **Du et al.** (theoretical, intake-569): position-inversion probability crosses 0.3 by "a few thousand tokens", → 0.5 by ~50K
- **Wang** (empirical, intake-547): depth-2 RLM accuracy degrades + wallclock inflates 96× (3.6s → 344.5s)

The depth-2 RLM aggregation step does long-context attention over the depth-1 children's outputs — exactly the regime Du et al. proved breaks down. **Wang's depth-2 cliff is mechanistically explained by Du's position-inversion proof.** This strengthens `01-fast-rlm-budget-controls.md`'s production depth-1 cap (now backed by theory + empirics, not just empirics).

### Autopilot digest

User explicitly approved adding option (b) "automated digest" from the three close-the-loop options. Built `scripts/autopilot/digest.py` (190 LoC) producing `progress/YYYY-MM/YYYY-MM-DD-autopilot.md` with:
- State snapshot: trial counter, session id, species budget, last digest date
- Pareto archive: entries, current hypervolume, slope (last 50 trials)
- StructuralLab summary: memory count, checkpoints, production-best flag
- Per NumericSwarm surface: completed trial count, best quality, cluster-selected best params (≥3 trials), fANOVA importance (≥10 trials)
- Recent trials (best-effort journal API probe)

**Append-only** (re-runs add stamped subsections, never overwrite). **Failure-tolerant** in the loop (try/except around `generate_digest` so digest issues never block optimization). **Hooked once-per-UTC-day** via `state["last_digest_date"]`. Also `python autopilot.py digest [--no-state-update]` for ad-hoc generation.

### Why NOT option (c) auto-handoff-update

Explicitly recommended against in this session. The autopilot controller is Claude CLI doing meta-reasoning — same hallucination/drift surface as any other Claude call. Auto-editing curated handoff content amplifies the cost of any miscall (overwritten caveats, premature promotions, edit drift between concurrent runs). Recovery is annoying (buried in git history). Digest gives 80% of the value at 10% of the risk by making manual review near-trivial.
