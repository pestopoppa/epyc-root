### Implementation details

- **Insertion point** is `stack_commands.cmd_start` line ~498 (after `--numa-mode` filter, before `[2] Checking target ports`). Chosen so prewarm scope tracks `--only`, `--include-warm`, and `--numa-mode` filtering exactly — never warms an unrelated WARM model.
- **Path resolution** reuses the same `build_server_command` dispatcher the launcher uses, then scans the resulting argv for `-m` / `-md` / `--mmproj`. This guarantees the warm set always matches the launch set without duplicating the role → path mapping (which is split across `validate_model_paths`, `WORKER_POOL_MODELS`, `VISION_*_MODEL`, etc.).
- **Inode dedupe**: `(st_dev, st_ino)` keys, not paths. Folds shared-GGUF roles (frontdoor's 5 quarters → 1 file, ingest's 5 → 1, etc.) onto a single warm operation. On the live stack: 39 server entries → 10 unique GGUFs.
- **Size-ordered warm** (largest first) so a small file's elapsed time isn't muddied by a big file still streaming in the kernel page cache.
- **Failure mode**: `prewarm_all` returns 1 if any warm fails, but the callsite continues startup. Warm failure is advisory — the launch path's own `mlock` will still pull pages from disk, just without the interleave guarantee.
