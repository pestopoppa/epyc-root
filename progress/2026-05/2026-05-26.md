## Session 16 — SSH tunnel for autopilot dashboard from travel laptop

### Trigger

Operator asked how to modify their travel laptop's `~/.ssh/config` so that running `ssh Beelzebub` would automatically expose the autopilot dashboard (`http://localhost:8000/dashboard/`) on the laptop browser. Recalled prior guidance involved an SSH proxy / port-forward setup.

### Resolution

Operator's existing config already does the two-hop bit (laptop → `aws-bridge` ProxyJump → reverse-tunneled `Beelzebub` on `aws-bridge:4444`). Only addition needed:

```sshconfig
LocalForward 8000 localhost:8000
ExitOnForwardFailure yes
```

Inside the `Host Beelzebub` stanza. The right-hand `localhost:8000` resolves on Beelzebub (final hop), which is correct because the orchestrator's dashboard binds `127.0.0.1:8000` server-side.
