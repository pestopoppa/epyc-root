## Fixes landed (in execution order)

### Item 1 — cmd_status / cmd_stop dict bug
**File**: `epyc-orchestrator/scripts/server/orchestrator_stack.py:820-832`

`load_state()` previously preserved non-ProcessInfo dict stubs (e.g. `{"port": 8000, "status": "preserved"}`), which then crashed downstream `info.pid` accessors. Now drops stubs with a single-line warning:

```python
out: dict[str, ProcessInfo] = {}
for k, v in data.items():
    if not isinstance(v, dict):
        continue
    try:
        out[k] = ProcessInfo(**v)
    except TypeError as exc:
        print(f"[load_state] dropping non-ProcessInfo entry {k!r}: {exc}")
return out
```

Verified: `status` and `stop --all` now run cleanly.

### Item 3 — gemma4 OMP idle-spin → KMP_BLOCKTIME=10
**File**: `epyc-orchestrator/scripts/server/orchestrator_stack.py:1944` (worker_pool branch, `if binary_override:` gate)

Investigation path:
1. Reverted the 2026-05-08 `OMP_WAIT_POLICY=passive` override (verified passive breaks MTP first-decode with `llama_decode ret=-3`).
2. Tried `omp_pause_resource(omp_pause_soft, ...)` then `omp_pause_resource_all(omp_pause_hard)` in `ik_llama.cpp/examples/server/server-context.cpp` at the `slots_idle()` transition — **both ignored by AOCC 5.0.0 libomp** (verified: 95+ threads stayed in R state with wchan=0 after the call).
3. Source patch reverted; `KMP_BLOCKTIME=10` set in launch env instead. LLVM libomp tunable that puts worker threads to sleep on a futex after 10 ms of idle. AOCC libomp respects it.

Wired in `start_server` worker_pool branch gated on `if binary_override:`, so any future ik_llama.cpp-based role inherits the fix automatically.

```

fix: gate TIDE dynamic early exit on explicit --n-layer-exit flag

Commit 0a9e8e5bc unconditionally activated TIDE layer reduction in the
server decode loop for all models. After 5 warmup tokens with 3 consecutive

so TIDE only activates when --n-layer-exit is explicitly passed.

Per `feedback_host_throttle_check`: "sustained multi-day load can degrade throughput ~60%". The `drop_caches` post-2026-05-06 fix did not restore. **Only untested lever: reboot.**
