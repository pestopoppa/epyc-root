## Session — research intake: Zensical (zensical.org)

### Context

User invoked `/research-intake https://zensical.org/` and then asked: "Many colleagues are migrating their documentations to it. What benefits would we get if we did the same?" Follow-up: deep-dive any repos.

### What Zensical is

- Open-source (MIT) static site generator from squidfunk (Material for MkDocs creator).
- Rust-core differential build engine (ZRX) + Python bindings. 4.7k stars, v0.0.43 alpha (2026-05-19), 446 commits.
- Existence is forced by MkDocs 2.0 dropping plugin support that Material relies on; Zensical is the squidfunk-blessed exit path. Material for MkDocs now in maintenance mode with 12-month critical-fix support.
- Headline: 4-5× faster *incremental* `serve` rebuilds (only changed files rebuild). Cold builds can REGRESS vs Material today (Python Markdown marshalling). Native `mkdocs.yml` read for drop-in migration. Bundles "Disco" client-side search.

### Relevance assessment — `not_applicable`

EPYC's documentation pipeline does NOT use MkDocs anywhere:

| Pillar | Reality | SSG? |
|---|---|---|
| `/workspace/wiki/` (29 files) | Hand-maintained markdown index | No |
| `epyc-inference-research/docs/chapters/` + `experiments/` | Raw markdown | No |
| `/workspace/handoffs/` | Raw markdown | No |
| Primary consumer | Claude agents via direct file reads + `kb-search` (ColBERT) | N/A |
| Secondary consumer | Humans reading via GitHub blob view | N/A |

The "4-5× faster builds" benefit presupposes an existing build step we don't have. ColBERT-backed `kb-search` already beats Disco for our actual primary user (agents). Migrating would *add* a build pipeline + deploy target, not optimize an existing one.

### Revisit triggers (when Zensical would jump to `worth_investigating`)

- Decision to publish a public-facing EPYC docs site (e.g., paper supplementary materials, open-sourcing the orchestrator).
- Onboarding outside collaborators who need a navigable docs surface.
- Zensical reaches 1.0 with full plugin parity AND a build need emerges.

Flagged via cross-ref to `handoffs/completed/knowledge-base-governance-improvements.md` as the natural revisit point.
