## REPL Prompt Overhaul & Routing Fixes

### Problem Identified
3-way eval showed SELF:direct passing while SELF:repl failing on the same questions (e.g., HellaSwag multiple-choice). Root cause: REPL mode forced code generation even for simple QA tasks.

### Research Conducted
Reviewed best practices from:
- [Anthropic Engineering](https://www.anthropic.com/engineering/writing-tools-for-agents): Tool descriptions need "when to use" AND "when NOT to use"
- [OpenAI Function Calling](https://platform.openai.com/docs/guides/function-calling): Explicit guidance on when not to use tools
- [ReAct Brittleness Paper](https://arxiv.org/abs/2405.13966): Performance depends on exemplar similarity
- Claude Code's own prompts: Conditional tool use, not forced code generation

### Changes Made

#### 1. REPL Prompt Overhaul (`src/prompt_builders/`)
**Before:**
```python
system="You are an orchestrator that generates Python code to solve tasks."
instruction="Write Python code to complete the task. Output only the code:"
```

**After:**
```python
system="You are an assistant that completes tasks. For simple questions
(factual, reasoning, multiple-choice), answer directly with FINAL(answer).
For tasks requiring file access, web search, or external data, use the available tools."
instruction="Complete the task. Output Python code with FINAL(answer):"
```

#### 2. Enriched Tool Descriptions (`src/prompt_builders/constants.py`)
Every tool now has 3-4 sentence descriptions including "when to use" and "when NOT to use":
```python
# Before
"- `web_search(query, max_results=5)`: Search the web"

# After
"- `web_search(query, max_results=5)`: Search the web via DuckDuckGo. Use ONLY when you
  need current/real-time info, or facts beyond your training data. Do NOT use for
  general knowledge, reasoning, or multiple-choice questions you can answer directly."
```

#### 3. Direct-Answer Examples Added
```python
## EXAMPLES: Direct Answer (NO tools needed)
Factual: `FINAL("Paris")`  # "What is the capital of France?"
Multiple choice: `FINAL("B")`  # "Which option is correct? A) ... B) ..."
Reasoning: `FINAL("The answer is 42 because...")`  # Math or logic problem
```

#### 4. Architect Delegation Fix (`src/prompt_builders/review.py`, `src/repl_environment/routing.py`)
- Changed architect delegation target from `coder_primary` to `coder_escalation`
- `coder_primary` = frontdoor (same model), delegation was pointless
- `coder_escalation` = Qwen2.5-Coder-32B with speculative decoding (different, capable model)
- Added `vision_escalation` to valid delegation targets
- Updated `_my_role()` to report correct delegation targets for Tier A/B

#### 5. Async Reward Injection (`scripts/benchmark/seed_specialist_routing.py`)
- Changed from blocking 10s timeout per reward to fire-and-forget async
- Uses ThreadPoolExecutor with 4 workers
- Each injection uses 30s timeout in background
- Eval loop no longer blocked by slow reward endpoint
