## Key Findings

1. Model-tier routing (10.8) is the dominant cost lever — present in 9/10 top Optuna trials.
2. Haiku handles tier-1 tasks at 78% cost reduction with no quality loss.

### Classifier-based routing confirmation (`confirm_20260224_010932`)
- Haiku pre-routing classifier reads task prompt, outputs HAIKU/SONNET/OPUS
- **Quality: 96.0% → 93.0% (−3.0pp regression) — Decision: DROP**
- **Cost: $0.0932 → $0.0786 per task (−15.7%)**
- Routing distribution: Haiku=47, Sonnet=46, Opus=7
- Classifier overhead was $1.83 (23.3% of B-arm cost) — CLI-invoked Haiku costs ~$0.018/call, much higher than the ~$0.001 API estimate
- `planning_synthesis` at 87% pass rate — classifier routed 19/31 to Haiku, too aggressive
- No improvement over static routing: same cost savings, worse quality, added overhead

### Abandoned approaches
- **Post-hoc Opus judging**: Haiku runs task → Opus judges pass/fail → escalate on FAIL. Abandoned because Opus judge input tokens cost ~$0.08-0.09/call, making judge overhead comparable to just running Opus directly.
- **Haiku-first with escalation chain**: All tasks to Haiku, escalate failures Haiku→Opus. Same issue — judge cost dominates.
