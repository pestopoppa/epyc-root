## Update: Debug Run Crash Investigation + Hardening

### Incident

During a live `--debug` seeding run, the process terminated with terminal output `Killed` shortly after ClaudeDebugger applied a fix and triggered orchestrator reload.

### Evidence

- `logs/debug_changes.jsonl` recorded a fix batch at `2026-02-21T14:44:49Z` with auto-commit and reload trigger.
- No matching new Python traceback was written to `logs/seeding_crash.log` for this incident.
- Symptom pattern is consistent with external process termination (non-Python unwind).

### Mitigation Implemented

| File | Change |
|------|--------|
| `scripts/server/orchestrator_stack.py` | Reload logic now kills by authoritative listener port instead of state-file PID for orchestrator and role-based reloads |
| `src/pipeline_monitor/claude_debugger.py` | Added structured reload watchdog logs: START / OK / FAIL / UNHEALTHY / EXCEPTION with elapsed timing and health endpoint |

### Operational Guidance

- `--debug` remains the live-fix mode (including reloads) and is still the intended interactive repair path.
- `--debug-dry-run` remains the safe observation mode when unattended stability is prioritized over live patching.

## Update: Nightshift Reliability Fix

### Incident Pattern

Nightshift was being triggered nightly, but recent runs were no-ops due to a missing worktree:
- `logs/nightshift/2026-02-18_023009.log`
- `logs/nightshift/2026-02-19_023000.log`
- `logs/nightshift/2026-02-20_023009.log`
- `logs/nightshift/2026-02-21_023009.log`

Each run aborted with:
- `ERROR: worktree not found at /mnt/raid0/llm/claude-nightshift`

### Fix Implemented

| File | Change |
|------|--------|
| `scripts/nightshift/run_wrapper.sh` | Added self-healing worktree bootstrap: prune stale entries, auto-create missing worktree (`origin/main` → `main` → `HEAD`), then proceed with normal sync/run |

### Outcome

Nightshift should no longer silently lose full nights due to missing worktree state. It will now attempt recovery automatically and only exit on unrecoverable git/worktree errors.
