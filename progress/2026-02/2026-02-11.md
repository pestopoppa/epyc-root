## Slot Erase Timeout Fix — SELF:direct Stale Generation

Fixed critical bug: SELF:direct timeout (600s) left server-side generation running, blocking SELF:repl and all subsequent strategies. TUI showed stale SELF:direct output indefinitely.

### Root Cause

1. `_erase_slots()` HTTP timeouts (3s) too short for busy server generating ~23k tokens
2. `_SLOT_ERASE_CAPABILITY` cache permanently disabled erase attempts after transient failures
3. Outer `except: pass` swallowed all diagnostic info
4. No proactive cancellation — relied solely on httpx timeout, by which point server was unresponsive to 3s erase calls

### Changes

| Item | Change | Impact |
|------|--------|--------|
| Erase HTTP timeouts | 3s → 8s | Busy servers can respond |
| `_force_erase_and_verify()` | New: resets cache, retries 3x with verification | Slot erasure actually works |
| Proactive timeout enforcement | Polling loop erases at `timeout - 15s` | Server stops before httpx timeout |
| Inter-strategy cleanup | Force-erase between SELF:direct → SELF:repl | Port is idle for next strategy |
| Diagnostic logging | `except: pass` → `logger.warning()` | Can debug erase failures |
| `all_slots` parameter | `_erase_slots(port, all_slots=True)` | Flush stale KV cache between questions |

## Seeding Script Bug Fixes (5 bugs from Feb 9-10 diagnostics)

Fixed 5 bugs identified from 1796 diagnostic entries: 120 duplicate entries, 5 double-timeout entries, 60 zero-token vision entries, 63 near_empty false positives, 65 unmitigated self_doubt_loops.

### Bug 1 (HIGH): Dedup race — questions re-processed up to 7x

`record_seen()` was only called when `rewards_injected > 0`. If injection failed, question stayed "unseen" and continuous-mode re-sampled it.

**Fix**: `seed_specialist_routing.py:462-467` — unconditional `record_seen()` after `checkpoint_result()`.

### Bug 2 (HIGH): Double timeout — retry uses same timeout budget

Retry path didn't distinguish timeout from transient connection errors. Timeout + retry = 2x elapsed (1200s instead of 600s).

**Fix**: `seeding_eval.py:231` — detect `"timed out"` / `"timeout"` / `"readtimeout"` in error message; skip retry for timeouts, only retry connection-type errors. Port cleanup (`_force_erase_and_verify`) still runs.

### Bug 3: Vision tokens_generated=0 (hardcoded)

`vision_stage.py:246-261` hardcoded `tokens_generated=0` and `tokens_used=0`. Vision backends don't expose completion token counts.

**Fix**: Estimate from `len(answer.split())` — word count proxy matches how `near_empty` uses it.

### Bug 4: near_empty false positives on short correct answers

Guard only checked `multiple_choice` and `exact_match` but missed `substring`, `f1`, `programmatic` — all can have legitimately short answers.

**Fix**: `anomaly.py:129-139` — invert logic: only fire `near_empty` for `code_execution` scoring.

### Bug 5: self_doubt_loop detected with no mitigation

65 detections but no mechanism to stop or redirect the model.

**Fix**: Added anti-self-doubt instruction to `frontdoor.md`, `architect_general.md`, `architect_coding.md` role prompts.
