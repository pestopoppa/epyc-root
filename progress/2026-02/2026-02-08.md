#### Phase 2A: Model Fallback with Cooldown (from OpenClaw)
- Added `FailoverReason` enum to `roles.py`: `circuit_open`, `timeout`, `connection_error`, `oom`.

- Feature flag: `session_compaction`.

### Architecture Decisions
1. **All flags default False** — zero behavioral change until explicitly enabled.
2. **Model fallback is distinct from task escalation** — fallback handles infrastructure failure (circuit open, timeout), escalation handles task complexity.
3. **ToolOutput slots into existing `Any` field** — no type changes needed for `ToolInvocation.result`.
4. **Cache skips non-deterministic requests** — only caches when `temperature == 0` and no custom stop sequences.
5. **Session compaction uses worker_summarize** (44 t/s) — cheap and fast enough for summaries.
6. **Resume tokens are compact** (<500 bytes) — JSON + zlib + base64url, no full prompt/context.
7. **Approval gates auto-approve by default** — `AutoApproveCallback` preserves current behavior.

## Streaming Inference Tap (Session 2)

### Summary
Implemented a real-time inference visibility layer activated by `INFERENCE_TAP_FILE` env var. When active, prompts and model output stream to a log file viewable via `tail -f`. Zero performance impact when disabled (O(1) env var check, no I/O). Intercepts at `_call_caching_backend()` in the inference path.

### Design
- **Activation**: `INFERENCE_TAP_FILE=/mnt/raid0/llm/logs/inference_tap.log`
- **Mechanism**: When active, `_call_caching_backend()` uses `infer_stream_text()` (SSE streaming with callback) instead of batch `infer()`. Each SSE text chunk is written to the tap file and flushed. Final SSE event extracts timings. Return type unchanged (`InferenceResult`).
- **Thread safety**: Module-level `threading.Lock` serializes all file writes.
- **Logrotate safe**: File opened/closed per `_append()` call — no stale handles.

## Rich TUI for Seeding Script (Session 3)

### Summary
Implemented a Rich split-screen TUI for the 3-way seeding script, activated via `--tui` flag. Single-command experience: left panel = seeding log, right panel = live inference stream (from tap file), bottom bar = progress status. Also added sentinel file mechanism so the API (separate process) discovers the tap path without env var propagation.

### Design
- **Left panel**: `DequeHandler(logging.Handler)` captures formatted records into `deque(maxlen=500)`. Replaces root logger handlers during TUI lifetime; restores on exit.
- **Right panel**: `TapTailer` daemon thread polls tap file at 50ms. Tracks current section between `========` markers. Handles file-not-yet-created gracefully.
- **Status bar**: `TUIProgress` dataclass updated via `on_progress` callback from `run_batch_3way()`.
- **Sentinel file**: TUI writes `/mnt/raid0/llm/tmp/.inference_tap_active` containing tap path. `inference_tap.py` checks sentinel as fallback (5-second cache, near-zero cost when disabled). Removed on exit + atexit safety.
- **Rich Live**: `SeedingTUI` implements `__rich_console__` protocol — layout rebuilt on every refresh tick (4 Hz).

## Heavy-Port Stall Forensics + Timeout Hardening (Session 6)

### Summary
Investigated why fresh 3-way runs still produced `INFRA` on heavy paths after label fixes. Root cause is mixed: (1) some calls are genuine timeout budget misses on long generations, (2) slot cleanup endpoint compatibility is inconsistent on current llama-server builds (`/slots/{id}?action=erase` often `501` or hangs), and (3) heavy-role throughput remains low (architect ports commonly ~3.6–5.8 t/s), so strict caps are fragile on hard prompts.

### Evidence Collected
- `benchmarks/results/eval/3way_20260208_171719.jsonl`:
  - `frontdoor:direct` consumed ~100.7s and 2048 tokens.
  - `frontdoor:repl` timed out at 179.1s with 0 tokens.
  - both architect variants timed out at ~420s with 0 tokens.
- `logs/llama-server-8083.log` / `logs/llama-server-8084.log`:
  - repeated long generations at ~3.6–5.8 t/s.
  - frequent `POST /slots/0` with `501`.
- `logs/llama-server-8080.log`:
  - repeated `POST /slots/0 501`, occasional cancel attempts while other tasks are still active.
- `logs/orchestrator_autolaunch.log`:
  - long lock spans on architect calls; confirms prolonged heavy-role occupancy windows.

### Changes Made
1. **Observed-runtime timeout bumping in 3-way eval**
   - Repl timeout is now raised from direct elapsed when needed.
   - Architect timeout is now raised from max(direct, repl) elapsed.
   - Keeps CLI `--timeout` as hard ceiling.
   - File: `scripts/benchmark/seed_specialist_routing.py`

2. **Slot erase capability detection + safe fallback**
   - Added per-port capability cache for slot-erase endpoint.
   - Stop logging false “erased slot” when endpoint is unsupported.
   - Disable repeated erase attempts after timeout/unsupported responses.
   - File: `scripts/benchmark/seed_specialist_routing.py`

## TUI Rendering Fixes + n_tokens Removal (Session 8)

### Summary
Fixed multiple Rich TUI rendering bugs discovered during live seeding runs, and removed the hard `n_tokens` generation cap across all roles — timeout is now the only hard limit. System prompts updated to steer models toward concise output.

### TUI Rendering Fixes
- **Rich Live `get_renderable` callback**: Replaced static renderable with `Live(get_renderable=self._make_layout)` — the correct way to display dynamic content in Rich Live (callback invoked fresh each tick).
- **Panel overflow fix**: Rich Panel renders from TOP and crops BOTTOM. Long log lines wrapped across multiple display lines, pushing newest messages off-screen. Fixed by calculating `panel_height` from terminal size and truncating each line to `panel_width` (half console width - 4 chars).
- **PROMPT filtering**: Right panel was flooded with full PROMPT sections (~30 lines each). Now filters PROMPT sections, replacing with `PROMPT: [...]`, showing only RESPONSE tokens and TIMINGS.
- **Status bar action extraction**: Bottom bar now extracts current action from last log record containing `→` (e.g., `SELF:direct (frontdoor:direct)`).
- **Logging noise**: Fixed `root.setLevel(logging.DEBUG)` capturing all third-party DEBUG messages. Changed to `logging.INFO` and silenced filelock, datasets, huggingface_hub, urllib3, fsspec.
- **Sentinel path hardcoding**: Fixed cross-process `$TMPDIR` mismatch by hardcoding `/mnt/raid0/llm/tmp/.inference_tap_active` in both TUI and inference_tap modules.
- **Chunk-based file reading**: TapTailer now uses `fh.read(8192)` with `_process_chunk()` instead of `readline()` to handle character-at-a-time SSE streaming.

### n_tokens → Unlimited (Timeout-Only Generation)
- **All `generation_defaults.n_tokens` set to `-1`** (unlimited) in `model_registry.yaml` — all 20+ roles including external API backends.
- **Default `n_tokens` changed from 512 to -1** in `InferenceRequest`, `backends/protocol.py`, and all `_real_call*()` methods.
- **Removed hardcoded n_tokens overrides** in `graph/nodes.py` (was 512 for compaction, 1024 for specialist calls).
- **Removed `n_tokens` from inference tap**: `tap_section()` and `write_header()` no longer accept/display `n_tokens`. Header now shows `ROLE=` only.
- **System prompt conciseness guidance added** to all role suffixes:
  - Frontdoor: "Be direct — avoid filler, repetition, and unnecessary preamble"
  - Coders: "No boilerplate or filler"
  - Ingest: "Be thorough but not verbose — prioritize signal over volume"
  - Architects: "stop once the decision/design is clear"
  - Thinking: "Reason carefully but stop once the conclusion is reached — no restating"
