## Session 7: CI Fix Completion (2538 tests passing)

### Problem

After Session 4's lint fixes, CI was still failing due to path-related issues. Tests that worked locally failed in CI because `/mnt/raid0/llm/claude/` doesn't exist in GitHub Actions runners.

Failing tests included:
- `TestRunTestsTool` tests (4 failures)
- Various REPL tests requiring local paths
- Registry loader tests requiring production model registry

### Root Causes

1. **Module-level `ALLOWED_PATHS`**: Computed at import time using `_get_project_root()`, which returns `/mnt/raid0/llm/claude` locally but the CI working directory in GitHub Actions
2. **Trailing slash mismatch**: Mock ALLOWED_PATHS used `["/mnt/raid0/llm/claude/"]` but resolved paths were `/mnt/raid0/llm/claude` (no trailing slash) — `startswith()` fails
3. **Missing fallbacks**: `_get_project_root()` in several files didn't fall back to `os.getcwd()` when the hardcoded path doesn't exist

### Fixes Applied

**test_tools_package.py** (TestRunTestsTool):
- Added `patch("src.tools.code.run_tests.ALLOWED_PATHS", ["/mnt/raid0/llm/claude"])` — no trailing slash
- Added `patch("src.tools.code.run_tests._get_project_root", return_value="/mnt/raid0/llm/claude")`
- Applied to 4 tests: `test_run_tests_success`, `test_run_tests_with_failures`, `test_run_tests_timeout`, `test_run_tests_with_pattern`

**Previous fixes in this session** (from summary):
- ProgressLogger: Fallback to temp directory when `/mnt/raid0/` not writable
- PDFRouter: try/except for temp_dir.mkdir() with fallback to system temp
- path_validation.py: Added `tempfile.gettempdir()` to allowed prefixes
- 6 files with `_get_project_root()`: Added `os.getcwd()` fallback
- test_repl_file_exploration.py, test_repl_file_mutation.py: Module-level CI skip
- TestProductionRegistry: CI skip decorator
- TestLintCodeTool: ALLOWED_PATHS mock (same pattern as run_tests)

## Session 9: CI Test Fix (MagicMock __str__ issue)

### Problem

CI run #21627707934 failed with `test_run_tests_success` asserting `False is True`. The test was trying to mock `Path` and configure `mock_dir.__str__.return_value`, but `str(MagicMock())` doesn't call `mock.__str__()` — Python calls `type(mock).__str__(mock)` which returns the default MagicMock string representation.

### Root Cause

```python
mock_dir = MagicMock()
mock_dir.__str__.return_value = "/mnt/raid0/llm/claude"
str(mock_dir)  # Returns "<MagicMock ...>" not "/mnt/raid0/llm/claude"!
```

Python's `str()` calls the dunder method on the **type**, not the instance. MagicMock's class-level `__str__` returns a mock representation string.

### Fix

Changed approach: mock `_validate_path` directly instead of trying to mock `Path`:

```python
# Before (broken):
with (
    patch("src.tools.code.run_tests.Path") as mock_path,
    patch("src.tools.code.run_tests.ALLOWED_PATHS", [...]),
):
    mock_dir.__str__.return_value = "/mnt/raid0/llm/claude"

# After (working):
with (
    patch("src.tools.code.run_tests._validate_path", return_value=(True, None)),
):
```
