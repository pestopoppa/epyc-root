### Problem

### Implementation

**New 3-Way Action Vocabulary:**
- `SELF:direct` - Frontdoor without tools
- `SELF:repl` - Frontdoor with tools, delegation disabled
- `ARCHITECT` - Architect with full delegation freedom
- `WORKER` - Scored via delegation chain attribution

**Key Design Decisions:**
- Binary rewards (1.0/0.0) for faithful P(success) estimation
- Cost is separate - stored in metadata for Optuna, not used in Q-value updates

## Session 23: Fix Missing 3-Way Checkpoint & Episodic Reset Script

### Problem

All 3-way eval answers were being lost. `run_batch_3way()` called `record_seen()` but never called a checkpoint function — the legacy `run_batch()` had `append_checkpoint()` but the 3-way code path was missing it entirely. This meant:

- `seen_questions.jsonl` was updated (questions marked as seen)
- But no session JSONL was written (answers, scores, timing all lost)
- Post-hoc debugging of failures was impossible

Discovered while investigating `thinking/hellaswag_20182` — a HellaSwag question (HellaSwag validation ind=20182) where all 4 modes (SELF:direct, SELF:repl, architect_general, architect_coding) returned FAIL with 0.0 reward. Without persisted answers, root cause analysis was impossible.

### Root Cause

`run_batch_3way()` (line 1607) only called:
```python
record_seen(qid, suite, session_id)  # marks as seen
# NO checkpoint — answers discarded!
```

The legacy `run_batch()` (line 1336) had both:
```python
append_checkpoint(session_id, result)  # saves answers
record_seen(result.question_id, ...)   # marks as seen
```

### Fix

Added `_checkpoint_3way()` function and wired it into `run_batch_3way()`:

```python
# Checkpoint immediately (answers, scores, timing for post-hoc analysis)
_checkpoint_3way(session_id, result)
# Record as seen
record_seen(qid, suite, session_id)
```

Session results now persist to `benchmarks/results/eval/{session_id}.jsonl` with full `ThreeWayResult` including all `RoleResult.answer` fields (truncated to 500 chars).

### Additional: Episodic Memory Reset Script

Created `scripts/session/reset_episodic_memory.sh` — safe reset that truncates SQLite tables and resets FAISS index in-place instead of deleting files (which breaks the running API's SQLite connection).
