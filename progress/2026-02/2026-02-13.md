## SoftMatcha v2 Research + Corpus-Augmented Speculative Decoding Plan

### Research Assessment

Reviewed SoftMatcha v2 (paper: arxiv 2602.10908, code: github.com/softmatcha/softmatcha2):
- **What it is**: Fast fuzzy pattern matcher for trillion-scale corpora. Python+Rust, suffix arrays + GloVe/FastText embeddings. <90ms on 6TB+, <1ms on ~100GB corpus (fits in RAM on our hardware). Apache 2.0.
- **Key capability**: Soft n-gram matching with word substitution/insertion/deletion tolerance, preserving word order.

### Critical Finding: 480B Prompt Lookup May Work

The model registry forbids prompt lookup on Qwen3-Coder-480B with reason "No prompt lookup - MoE architecture". **This is likely wrong:**
- Prompt lookup proposes drafts from the same model's tokenized prompt ‚Äî no cross-model BOS mismatch
- MoE expert reduction is orthogonal to the token proposal/verification loop
- The SSM ban (Qwen3-Next) is correctly justified (consecutive position requirement), but MoE has no such constraint
- `run_combination_benchmarks.sh:213` already has a `run_lookup_hard_mask` entry for 480B that was set up but never executed

**Potential impact**: If prompt lookup works on 480B code editing tasks, MoE3 (10.3 t/s) + lookup could reach 15-25+ t/s. If jukofyork draft also works, triple-stacking (MoE3 + spec + lookup) could reach 25-35 t/s from a 480B-class model.
