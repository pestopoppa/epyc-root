# Progress Report — 2026-02-12

## `repl_no_tools` Signal Fix (A + B + C)

`repl_no_tools` is the #2 anomaly signal (159 hits, 7.6%) from the 40-hour seeding debug run. Three sub-problems addressed:

### Fix A: Smarter Direct-Mode Routing

Simple factual/MCQ questions were routed to REPL mode, where models answered directly without tools — triggering the `repl_no_tools` signal even though behavior was correct.

| Item | Change |
|------|--------|
| `_should_use_direct()` | New heuristic: MCQ (3+ choices, <2000 chars) or short factual (<300 chars, question-word prefix) → direct mode |
| `_select_mode()` | Heuristic runs before MemRL check; REPL keywords/long context/coding tasks always fall through to REPL |
| Conservative scope | Only short-circuits obvious cases; MemRL can still override |

### Fix B: Max-Turns Answer Rescue

Models that reasoned correctly but never called `FINAL()` within 10 turns lost their answer completely.

| Item | Change |
|------|--------|
| `_rescue_from_last_output()` | New helper: tries FINAL() extraction → prose answer → code block extraction from `state.last_output` |
| Graph node max-turns | All 7 node classes (Frontdoor, Worker, Coder, CoderEscalation, Ingest, Architect, ArchitectCoding) try rescue before returning `[Max turns]` error |
| Post-graph fallback | `repl_executor.py` tries rescue before generating `[Max turns ... without FINAL()]` error message |

### Fix C: Graduated Turn Nudge

Hard deadline at `remaining <= 3` caused panic responses. Added earlier soft nudge.

| Turns Remaining | Nudge |
|----------------|-------|
| `max_turns // 2` (midpoint, e.g. turn 5/10) | Soft: "Start converging on your answer. Call FINAL() when ready." |
| `<= 3` (existing) | Hard: "You MUST call FINAL() NOW. Do NOT start over." |
