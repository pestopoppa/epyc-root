## Overnight Run Regression Fixes

### Summary

Fixed 4 bugs exposed by overnight `seed_specialist_routing.py --evolve --debug-replay --continuous` run. All regressions caused by interaction between new features (corpus injection, skill evolution, nightshift) and existing delegation/seeding infrastructure.

### Fixes Applied

| Fix | Severity | Root Cause | Impact |
|-----|----------|------------|--------|
| Escalation loop guard | CRITICAL | Brief dedup used first 200 chars (architect rephrasing defeats it); no role repetition tracking; unbounded re-entrance | 4 new guards: semantic dedup (hash brief+target), thread-local depth counter, role repetition (max 2 consecutive), token budget (20K cap) |
| Corpus injection in delegation | CRITICAL | `_run_specialist_loop()` called `build_root_lm_prompt()` without `corpus_context=`; delegated 32B/480B never received corpus | Added `build_corpus_context()` call on turn 0, passed to `build_root_lm_prompt()` |
| `--evolve` / `--debug-replay` silent no-op | CRITICAL | OutcomeTracker gated by `ORCHESTRATOR_SKILLBANK=1` env var; evolve/replay blocks sit after continuous `while` loop (only run on Ctrl+C exit, never on SIGTERM) | Removed env var gate; added `_run_post_batch_hooks()` every 10 batches |
| `no_skills_available` false positive | LOW | `skill_coverage = (skills_retrieved >= 0)` always True; `skills_retrieved` default 0 indistinguishable from "SkillBank not loaded" | Changed default to `None`; `skill_coverage` now True only when SkillBank actually initialized |

## Distillation Pipeline Latency Instrumentation

### Problem

TUI showed visible latency spikes during prompt transitions between models in the seeding script. `ClaudeDebugger` could not detect this: `slow_delegation` signal threshold is 120s (designed for catastrophic stalls), and it only fires on orchestration graph delegation events, not teacher HTTP calls within distillation.

### Fixes Applied

| Fix | File | Impact |
|-----|------|--------|
| httpx client reuse | `teachers.py` | Eliminates per-batch TCP connection setup (~50-200ms overhead per batch). Lazy init + async context manager. |
| Per-batch timing | `pipeline.py` | New `batch_latencies` on `DistillationReport`: `{skill_type, batch_index, batch_size, elapsed_ms, teacher}`. Logged at INFO. |
| `distill_batch_latency` signal | `anomaly.py` | Fires at 5s threshold (vs 120s). Weight 0.5. Backward-compatible kwarg in `compute_anomaly_signals()`. |

### Files Changed

- `orchestration/repl_memory/distillation/pipeline.py` — batch timing instrumentation + report field
