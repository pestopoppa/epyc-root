## Overnight Debugger Analysis & Pipeline Fixes

### Summary

Analyzed 706 diagnostics and 270 debug changes from 18h overnight seeding run. Fixed two retry mechanism bugs, expanded anomaly detection from 12 to 17 signals, added auto-discovery mechanism for new failure patterns, and optimized architect delegation routing.

### Bug Fixes

1. **Retry timing race**: `pop_retries()` used non-blocking `_collect_background()` — Claude subprocess always still running → retries never fired. Fixed to blocking `_wait_background()`.
2. **Retry state loss**: 94 session restarts wiped in-memory `_retry_queue`. Added JSONL persistence to `logs/retry_queue.jsonl` — survives crashes.

### New Anomaly Detectors (5)

| Signal | Weight | Description |
|--------|--------|-------------|
| `repl_no_tools` | 0.5 | REPL mode with zero tool calls |
| `slow_delegation` | 0.5 | Any delegation hop >120s |
| `function_repr_leak` | 1.0 | `<function foo at 0x...>` in answer |
| `status_phrase_final` | 1.0 | "Done"/"Complete" as final answer |
| `misrouted_to_coder` | 1.0 | Factual/MCQ delegated to coder |

### Auto-Discovery Mechanism

Debugger now instructs Claude to propose new signal detectors via structured `NEW_SIGNAL:` format. Proposals persisted to `logs/proposed_signals.jsonl` for human review.

### Architect Routing Optimization

Rewrote `architect_investigate.md` to prevent factual questions being misrouted to coder_escalation (architect_coding path was 4x slower: median 58s vs 13s).

### Seeding Script Transition

Confirmed monolithic → refactored module parity. All 28 CLI flags, all modes, debugger integration verified. Ready for transition.
