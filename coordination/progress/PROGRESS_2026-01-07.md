## Performance Tuning Research (Session 5)

**Time**: End of day continuation
**Status**: ✅ RESEARCH COMPLETE

### Summary

Researched inference optimization techniques for orchestrator performance tuning. Identified training-free approaches that can be applied without model downloads or fine-tuning.

### Verbalized Sampling (VS) - Uncertainty Quantification

**Paper**: https://arxiv.org/abs/2510.01171 (Ramirez et al., Nov 2024)
**GitHub**: https://github.com/CHATS-lab/verbalized-sampling
**License**: Apache 2.0

**Key Insight**: RLHF-trained models have collapsed diversity - they confidently produce single answers even when uncertain. VS recovers this diversity by prompting the model to generate k responses with probabilities.

**Prompt Template**:
```
Generate 5 responses to the user query, each within a separate <response> tag.
Each <response> must include a <text> and a numeric <probability>.
Please sample at random from the tails of the distribution, such that
the probability of each response is less than 0.10.
```

**Results from Paper**:
| Dataset | Self-Consistency | Self-Evaluation | Verbalized Sampling |
|---------|------------------|-----------------|---------------------|
| GSM8K | 88.6% | 88.7% | **94.1%** |
| MATH | 39.7% | 43.5% | **53.1%** |
| MMLU | 79.6% | 80.1% | **83.1%** |

**Proposed Use Case for Orchestrator**:
- When frontdoor receives ambiguous task, use VS instead of temperature tuning
- If top probability >> others → proceed with deterministic routing
- If probabilities close → surface ambiguity to user or try multiple approaches
- Trade-off: ~5x output tokens, but training-free and recovers lost diversity

### Temperature Tuning Analysis

**Trade-offs Analyzed**:

| Temperature | Pros | Cons |
|-------------|------|------|
| T=0.0 | Deterministic, reproducible, max parse success | Stuck in local optima, no recovery from errors |
| T=0.1 | Near-deterministic, slight exploration | Good balance for structured output |
| T=0.3 | Moderate diversity | May cause parse failures on strict JSON |
| T=0.5 | High diversity, creative tasks | Unreliable for orchestration IR |

**Recommendation**: Test range T ∈ {0.0, 0.1, 0.3, 0.5} empirically, measure parse success rate.

### Early Failure Detection Heuristics

**Source**: `research/early_failure_prediction.md`
**Status**: Training-free, ready to implement

| Heuristic | Description | Threshold (Worker) |
|-----------|-------------|-------------------|
| High entropy | Token distribution too flat | >4.5 |
| Entropy spike | Sudden uncertainty increase | >2.5 delta |
| Repetition | N-gram repetition ratio | >0.2 |
| Early check | Checkpoint tokens | 50, 100 |

**No downloads required** - uses token-level statistics from generation.
