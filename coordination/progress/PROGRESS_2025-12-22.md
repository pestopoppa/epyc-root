## 1. MOE2 Speed Investigation

### Problem
Benchmark showed MOE2 at ~5.69 t/s (0.83x baseline) when manual tests showed ~12 t/s (1.8x baseline).

### Investigation

| Test | Speed | Notes |
|------|-------|-------|
| MOE2 (manual, no flash attn) | 12.10 t/s | Correct behavior |
| MOE2 (manual, with flash attn) | 12.10 t/s | Flash attn doesn't affect gen speed |
| MOE2 (old benchmark) | 5.69 t/s | Wrong - ran before fix |

### Root Cause
The MOE2 benchmark results (timestamp: 2025-12-21T06:31:07) were run **BEFORE** the server restart fix was committed (2025-12-21 12:15:31).

All MoE configs (MOE2, MOE4, MOE6, MOE8) ran on the same server that was started with default 8 experts. The "MOE2" results were actually 8-expert runs, explaining why speeds matched baseline.

### Verification
- Server logs confirmed override: `Using metadata override (int) 'qwen3moe.expert_used_count' = 2`
- Server correctly shows: `n_expert_used = 2`
- Manual test with proper 2-expert server: 12.10 t/s

### Resolution
Benchmark needs re-run with corrected server restart logic to get accurate MoE comparison data.
