## Components Implemented

### 1. REPL Environment (`src/repl_environment.py`)

Sandboxed Python REPL implementing the RLM (Recursive Language Models) pattern where context is treated as an external object.

**Features:**
- Context injection as `context` variable
- `peek(n)` - Inspect first n chars of context
- `grep(pattern)` - Regex search in context
- `FINAL(answer)` / `FINAL_VAR(var_name)` - Completion signals
- Output capping at 8192 chars
- Configurable timeout (default 120s)

**Security Sandbox:**
Blocks dangerous operations via code validation:
- `import os`, `import subprocess`
- `__import__`, `eval()`, `exec()`, `compile()`
- `open()`, `getattr()`, `setattr()`
- `__class__`, `__subclasses__`, `__globals__`

**Tests:** 49 passing

### 2. LLM Primitives (`src/llm_primitives.py`)

Sub-LM spawning functions for orchestration.

**Features:**
- `llm_call(prompt, context_slice, role)` - Single sub-LM call
- `llm_batch(prompts, role)` - Parallel sub-LM calls
- Mock mode for testing without model execution
- Call logging for debugging
- Output capping per call
- Statistics tracking

**Role routing:** worker, coder, ingest, architect

**Tests:** 31 passing

### 3. Gate Runner (`src/gate_runner.py`)

Automated quality gate execution with structured failure parsing.

**Features:**
- Load gates from `config/gates.yaml`
- Run individual gates or all gates
- Parse output for errors and warnings
- Timeout handling per gate
- Retry support
- Summary generation

**Configured Gates (7):**
| Gate | Timeout | Required |
|------|---------|----------|
| schema | 10s | No |
| format | 30s | Yes |
| lint | 60s | Yes |
| typecheck | 120s | No |
| unit | 180s | Yes |
| integration | 300s | No |
| shellcheck | 30s | No |

**Tests:** 22 passing

---

### 4. Failure Router (`src/failure_router.py`)

Escalation chain routing for failed tasks.

**Escalation Chains:**
```
worker → coder → architect → FAIL
ingest → architect → FAIL
```

**Features:**
- Configurable retry counts per role
- Error category handling (code, logic, timeout, schema, format)
- Optional gate timeout skipping
- Escalation history tracking
- Formatted failure reports

**Special Rules:**
- Format/schema errors: retry only, never escalate
- Timeout on optional gate: skip gate
- Max escalations enforced per chain

**Tests:** 51 passing

### How to Use

**1. Add to PATH (each session or in shell config):**

```bash
export PATH="/mnt/raid0/llm/npm-global/bin:/mnt/raid0/llm/tools/devc/bin:$PATH"
```

**2. Use devc commands:**

```bash
# Full workflow: install template + start container + attach tmux
devc /path/to/repo

# Install template only (creates .devcontainer/ in repo)
devc install /path/to/repo

# Rebuild container (clears cache, keeps volumes)
devc rebuild /path/to/repo

# Execute command in container
devc exec /path/to/repo -- <command>
```

### Prerequisites

**Docker group membership required** (one-time setup):
```bash
# Add yourself to docker group
sudo usermod -aG docker $USER

# Apply immediately (or log out and back in)
newgrp docker
```

Without this, you'll get: `permission denied while trying to connect to the docker API`
